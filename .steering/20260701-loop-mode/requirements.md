# 要求内容

## 概要

全ステージクリア後に「次の周回へ」進めるNew Game+的な仕組みを追加し、周回ごとに敵が段階的に強化される・演出が変化する・見た目の報酬が得られるようにすることで、クリア後のリプレイ性を高める。

## 背景

直近の開発はビジュアル演出・バランス調整・モバイル対応が中心で、クリア後の遊びは「タイトルへ戻る／任意ステージ再挑戦」に留まっている。セーブ構造(`clearedStages`, `bestTimeMs`, `difficulty`, `busterMode` を持つ `SaveData`)や難易度係数(`src/config/balance.ts`, `src/systems/difficulty.ts`)、hard専用コンテンツ(`ShadowRayBoss`)など、周回要素を載せる土台はすでに整っている。この土台を活用し、低コストで「もう一周したくなる」体験を追加する。

## 実装対象の機能

### 1. 周回カウントの永続化
- `SaveData` に `loopCount: number`(初期値1)を追加し、`localStorage` に保存する。
- 既存セーブ(v4以前)からのマイグレーションで `loopCount: 1` を補完する。

### 2. 周回への遷移フロー
- 最終ステージ(stage6)クリア後、既存のエンディング演出の後に「次の周回へ進む」選択肢を提示する画面/ダイアログを追加する。
- プレイヤーが選択したら `loopCount` をインクリメントし、`clearedStages` をリセットして stage1 から再開できる状態にする(セーブデータのベストタイム等、周回に関係しない記録は保持)。
- 選択しない場合は現行どおりタイトルへ戻る。

### 3. 段階的な難易度スケーリング
- 周回数に応じて既存の hard 係数(`enemyHpFactor`, `enemySpawnMultiplier`, `playerDamageMultiplier`, `turretIntervalFactor`, `walkerSpeedFactor` 等)にさらに乗数を掛ける仕組みを `src/config/balance.ts` 系に追加する。
- スケーリングは3周目で上限に達し、以降(4周目以降)は数値を頭打ちにする(青天井にしない)。
- 選択中の難易度(normal/hard)の係数に対して、周回スケーリングを重ねて適用する。

### 4. 2周目以降のストーリー演出スキップ
- hard modeの既存スキップ実装(開始/救出/エンディング演出の非表示)を流用し、`loopCount >= 2` の場合も同様にスキップする。
- normal/hard を問わず、2周目以降は演出が省略される。

### 5. 見た目の報酬
- RAYの色バリエーション: 既存の `ShadowRayBoss` の赤紫反転パレットの仕組みを流用し、周回数に応じてプレイヤーキャラRAYの配色を変更できるようにする(周回ごとに固定の配色を割り当てる想定)。
- タイトル画面演出の変化: 周回数に応じてタイトル画面の背景/エフェクトを変化させる。

## 受け入れ条件

### 周回カウントの永続化
- [x] 新規セーブでは `loopCount: 1` から開始する。
- [x] 既存セーブ(v4以前)を読み込んだ場合、`loopCount: 1` として自動補完される。
- [x] 周回が進むたびに `loopCount` が保存され、リロード後も保持される。

### 周回への遷移フロー
- [x] stage6クリア後のエンディング演出の後、「次の周回へ進む」を選べる導線が表示される。
- [x] 「次の周回へ進む」を選択すると `loopCount` が+1され、`clearedStages` がリセットされ、stage1から再開できる。
- [x] 選択しない場合は現行どおりタイトル画面に戻る。

### 段階的な難易度スケーリング
- [x] 2周目・3周目でそれぞれ敵HP/被ダメージ等の係数が段階的に上昇することを確認できる。
- [x] 4周目以降は3周目の係数から変化しない(上限固定)。
- [x] normal/hardどちらの難易度でも周回スケーリングが正しく重ねて適用される。

### 2周目以降のストーリー演出スキップ
- [x] `loopCount >= 2` の場合、難易度がnormalであっても開始/救出/エンディング演出がスキップされる。
- [x] 1周目は既存どおり難易度に応じた演出表示/非表示が維持される。

### 見た目の報酬
- [x] 周回数に応じてRAYの配色が変化し、プレイ画面で視認できる。
- [x] 周回数に応じてタイトル画面の演出(背景/エフェクト)が変化する。

## 成功指標

- 定性: プレイテストで「クリア後もう一周したくなった」というフィードバックが得られる。
- 定量: (任意)特に無し。まずは仕組みの導入を優先する。

## スコープ外

以下はこのフェーズでは実装しません:

- 実績システム、スコアランキングなど今回選ばなかった他のリプレイ性強化案。
- 周回ごとの新規ステージ/新規ボス追加。
- 周回報酬としてのアイテム・スキルなどゲームプレイに影響する要素(見た目のみに留める)。
- 4周目以降のさらなる難易度追加(段階は3周目で頭打ちにする)。

## 未決事項

- 「次の周回へ進む」導線の具体的なUI(専用画面か、既存のClearScene/タイトル画面に組み込むダイアログか)は設計フェーズで決める。→ 既存の `ClearScene`(ALL CLEAR画面)を拡張する形で決定・実装済み。
- RAY配色バリエーションの具体的な色指定(周回2=何色、周回3=何色等)は設計/実装フェーズでビジュアル調整として決める。→ プレースホルダ配色(`loopRayTint`)で実装済み。正式な色は別途ビジュアル調整タスクで確定する。
- 周回スケーリングの具体的な乗数値(2周目・3周目それぞれどの程度強化するか)はバランス調整として実装フェーズで決める。→ `loopScaling` テーブルとして実装済み。実プレイでの体感バランス調整は今後の課題として残る。
