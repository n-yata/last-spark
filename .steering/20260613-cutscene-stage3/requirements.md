# 要求内容

## 概要

演出シーン基盤（静止画＋交互テキスト＋ト書き）を実装し、それを使って Stage 3（収容施設）を追加する。Stage 3 は TERRA の初登場・救出という物語の転換点であり、ボス撃破後の演出シーンを初めて導入する。あわせて SaveData を全6ステージ対応に拡張する。

## 背景

ブロック1でテキスト表示・ログ基盤が整う。Stage 3 では TERRA との出会いを「演出シーン」（ゲームを止めてキャラのセリフとRAYの内心を交互に見せる）として描く必要があり、これは Stage 4-6 でも使う共通基盤になる。また Stage 3 の追加で初めて「ステージが3つ以上」になるため、クリア状況を1フラグで持つ現行 SaveData を多ステージ対応にする必要がある。

## 実装対象の機能

### 1. 演出シーン基盤（CutsceneSystem）

- ゲームを一時停止し、静止画的な簡易演出の上にテキストを順番に表示する仕組み
- `docs/story.md` の演出スクリプト（TERRAのセリフとRAYの内心の交互表示）を再生できる
- ト書き（「TERRAがRAYの胸部の刻印に気づく」等）を画面中央に状況説明テキストとして表示する
- タップで次のテキストへ送り、全テキスト後にゲームプレイ／次シーンへ遷移する
- ステージ開始演出・ボス後演出の両方に対応する

### 2. Stage 3（収容施設）

- ステージデータ（地形・敵・ログトリガー3本・ボストリガー）
- ボス「収容番人（重装型）」: 遅いが重い単発攻撃の接地型
- 収容ケージのギミック（ボス撃破で解錠される演出）
- 救出後演出シーン（TERRA初登場・名乗り・刻印で名前が判明）
- Stage 3 確定テキスト（開始テキスト・ログ3本・語りかけ・内心・TERRAセリフ）の組み込み

### 3. SaveData の全6ステージ対応

- クリア状況を「ステージ単位」で持てるよう拡張する
- 既存セーブからのマイグレーション（version 更新）
- 進捗に応じて到達済みステージから再開／表示できる素地を作る

### 4. ボス撃破後フローの拡張

- 現行「ボス撃破→即クリア遷移」を「ボス撃破→ボス後ログ→ボス後演出シーン→クリア処理」に拡張する
- Stage 1-2 は従来どおり（ボス後演出なし）でも動くよう後方互換を保つ

## 受け入れ条件

### 演出シーン基盤
- [ ] ゲームを一時停止し、交互テキスト（話者でスタイルが変わる）を順に表示できる
- [ ] ト書きが状況説明として表示される
- [ ] タップ送りで進み、最後にゲーム／次シーンへ遷移する
- [ ] 開始演出・ボス後演出の両方で使える

### Stage 3
- [ ] Stage 3 に到達でき、収容番人ボスと戦える
- [ ] ボス撃破で収容ケージが解錠される演出が出る
- [ ] 救出後演出で TERRA が登場し、刻印で RAY の名前が判明する確定スクリプトが再生される
- [ ] Stage 3 の開始テキスト・ログ3本・語りかけ・内心が確定版どおり表示される

### SaveData
- [ ] クリア状況がステージ単位で保存される
- [ ] 既存セーブが壊れず、マイグレーションで引き継がれる
- [ ] SaveManager のテストが追加・更新されている

### ボス後フロー
- [ ] Stage 3 でボス撃破→ログ→演出→クリアの順で流れる
- [ ] Stage 1-2 が従来どおり動く（後方互換）

### 品質
- [ ] `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` が通る

## 成功指標

- Stage 3 をプレイして、TERRA 救出が物語の転換点として感情的に伝わる
- 演出シーン基盤が Stage 4-6 でそのまま再利用できる状態になる

## スコープ外

- Stage 4以降のステージ・ボス
- BGM 拡張 → ブロック6
- TERRA のゲームプレイ中の同行表示（設定上は安全エリア待機で画面に出ない）
