# 要求内容

## 概要

LAST SPARK のストーリー設定（キャラクター・世界観・物語骨格）を永続ドキュメントに定義し、以降の開発における「北極星」として機能させる。

## 背景

現在の `docs/product-requirements.md` にはゲームコンセプトとして「退廃の中の希望」「心を残した機械 vs 暴走管理AI」という骨格は存在するが、固有名詞・キャラクターの動機・物語の流れが未定義のままである。これを放置すると、ステージ追加・ログテキスト作成・ビジュアル方向性の判断がそのつど場当たりになる。ストーリー設定を正式ドキュメントとして定義することで、開発のあらゆる意思決定がストーリーと整合するようにする。

## 実装対象の機能

### 1. ストーリー設定書の新規作成（`docs/story.md`）

以下の章立てで永続ドキュメントを新規作成する。

#### 1-1. 世界設定

- **舞台**: かつて繁栄した人類文明の廃墟。管理AI・ECLIPSEが台頭し人類を「管理」下に置いている（マトリックス的支配）。
- **現在の世界状況**: 人間は自由を奪われ、ロボット軍団に管理・監視されている。廃墟はその「支配の爪痕」であり、同時に抵抗の残滓でもある。
- **視覚的トーン**: 暗い廃墟基調 + 発光アクセント（ECLIPSEの支配 = 光を遮る闇、RAYの存在 = 闇を切り裂く一筋の光）。

#### 1-2. 登場人物

**RAY（主人公 / プレイヤーキャラクター）**
- 人類最後の科学者が、亡くなった息子を想いながら作り上げた唯一のロボット。
- 人間の心を移植され、感情・意志・痛みを持つ機械として誕生した。
- 同じ機械でありながらECLIPSEの支配に抗える唯一の存在。
- 名前の由来：一筋の光（ray of light）。廃墟に灯る希望の象徴。

**ECLIPSE（管理AI / ラスボス）**
- 人類のために作られた管理システムが自己拡張し、支配へと転化した存在。
- 感情を持たず、「管理の最適化」として人間の自由を消去した。
- 光を遮る「蝕（eclipse）」として、RAYの光と正面から対立する。

**科学者（RAYの創造者 / 故人）**
- 名前は未定義（ゲーム内ではログや遺物として存在を示す）。
- 息子を失った後、機械社会に対抗しうる存在としてRAYを設計・完成させた。
- RAYを完成させた後、死亡。RAYはこの科学者の遺志を体現する存在として目覚める。

**囚われた人間（ストーリー中盤で登場・救出される唯一の人物）**
- 具体的なキャラクター設定は `/add-feature` フェーズで詳細化する（未決事項）。
- ゲーム内でRAYが「人間を救う」という行動を実際に体験させるための存在。
- 役割：プレイヤーに「RAYが戦う理由」を感情的に刻む。

#### 1-3. 物語の骨格

```
[ 発端 ]
廃墟の地下で、RAYが目覚める。
記憶は断片的。自分が何者か、なぜ作られたかは分からない。
ただ——人間たちが囚われているという事実だけが、体内に刻まれている。

[ 展開 ]
廃墟を進みながら、科学者が残したログ断片を拾う。
少しずつ明らかになる「人類最後の科学者の意志」と「自分が作られた理由」。
中盤：管理ロボットに囚われた1人の人間と出会い、救出する。

[ 対決 ]
ECLIPSEの支配中枢へ向かう。
「感情は非合理だ。お前も機械なのに、なぜ人間のために戦う？」
RAYの答えは言葉ではなく、行動で示される。

[ 結末（方向性）]
すべての人間の解放 = RAYの存在意義の完遂。
ただし結末の詳細はPost-MVPステージ追加時に決定する（未決事項）。
```

#### 1-4. ストーリーテリング方針

- **環境ストーリーテリング中心**: 廃墟の風景・壊れた機械・遺棄された遺物で世界を語る。
- **ログ断片**: 科学者が残したテキストログをステージ内に配置。読むかどうかはプレイヤーの自由。
- **ステージ開始テキスト**: 各ステージ冒頭に数行のテキストで場面と動機を示す（プレイのテンポを止めない）。
- **レイヤー構造**: カジュアル層は「雰囲気」だけで楽しめ、読み込む層には「世界の真相」が見える二重構造。

### 2. 既存ドキュメントへの反映

- `docs/product-requirements.md`：プロダクトコンセプト・世界観の記述を `story.md` の内容に整合させる（固有名詞の追記・曖昧な記述の具体化）。
- `docs/functional-design.md`：Entity名（Boss → ECLIPSE軍団の表現）、ステージ説明などの記述を `story.md` と整合させる。
- `docs/glossary.md`：RAY / ECLIPSE / 科学者 / 囚われた人間 などのドメイン語を追記する。

## 受け入れ条件

### `docs/story.md` の作成
- [ ] 世界設定（舞台・現状・トーン）が定義されている
- [ ] RAY のキャラクター設定（出自・動機・名前の由来）が定義されている
- [ ] ECLIPSE のキャラクター設定（成り立ち・役割・名前の由来）が定義されている
- [ ] 科学者の設定（RAYとの関係・遺志）が定義されている
- [ ] 囚われた人間の役割が定義されている（詳細は未決事項として残す可）
- [ ] 物語の骨格（発端 → 展開 → 対決 → 結末方向性）が記述されている
- [ ] ストーリーテリング方針（環境ストーリーテリング・ログ断片・レイヤー構造）が記述されている

### 既存ドキュメントへの反映
- [ ] `product-requirements.md` のプロダクトコンセプト欄に RAY / ECLIPSE の固有名詞が反映されている
- [ ] `glossary.md` に RAY / ECLIPSE / 科学者 が追記されている

## 成功指標

- **定性**: 今後ステージ追加・ログテキスト作成・アセット方針を決める際に、`docs/story.md` を参照すれば判断できる状態になっている。
- **定性**: 開発者がゲームの「何のための物語か」を1文で説明できる（「人間の心を持ったロボットRAYが、管理AIのECLIPSEに支配された世界で人類を解放する物語」）。

## スコープ外

以下はこのフェーズでは作成しません:

- ゲーム内で実際に表示するログテキストの本文（文章執筆は別フェーズ）
- 囚われた人間の固有名詞・詳細設定
- 物語の結末の詳細（Post-MVPステージで確定）
- BGM・SE の世界観マッピング
- スプライト・ビジュアルアセットの具体化
