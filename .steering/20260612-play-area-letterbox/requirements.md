# 要求内容

## 概要

タッチ操作時、画面下部に仮想ボタン専用の帯（レターボックス）を設け、ゲーム描画領域を
その上に収める。指が乗るボタン帯とプレイ領域を物理的に分離し、ボス戦などで指が
ボスや弾幕を隠す問題を解消する。

## 背景

ボスはアリーナ右側に出現する一方、仮想ボタン（JUMP / SHOT）は画面右下角にあるため、
横持ちで右親指がボタンに乗ると指そのものが右下のボス・弾幕を隠してしまう。ボタンは
半透明のため透明度調整では解決できず、「指が乗る場所」と「見える場所」を分離する必要がある。

## 実装対象の機能

### 1. 下部ボタン帯（レターボックス）の導入

- タッチ環境では、画面下部に一定高さの専用帯を確保し、その上にゲーム描画領域を収める
- ゲームのワールド表示（カメラ viewport / zoom）を帯の高さぶん上側へ詰める
- 全ステージで常時適用する（ボス戦に限定しない）

### 2. 仮想ボタン・移動ゾーンの帯内再配置

- JUMP / SHOT ボタンと移動ゾーンのヒント類を、新設した下部帯の中に収める
- 入力判定（InputController）と描画（TouchControls）のレイアウトを帯基準で一致させる
- RESIZE / 画面回転でも帯とボタン位置が破綻しない

### 3. デスクトップ（非タッチ）での従来挙動維持

- タッチ非対応環境（キーボード操作）では帯を出さず、従来通りフル画面でプレイできる

## 受け入れ条件

### 下部ボタン帯

- [ ] タッチ環境で、画面下部に仮想ボタン専用の帯が表示される
- [ ] ゲームのプレイ領域（ワールド描画）が帯と重ならず、帯の上側に収まる
- [ ] ボス戦でボス本体・弾幕が帯の上のプレイ領域に表示され、ボタンに乗せた指で隠れない
- [ ] 全ステージ（通常進行〜ボス戦）で帯が常時表示される
- [ ] 画面リサイズ・縦横回転を行っても、プレイ領域と帯の比率・ボタン位置が破綻しない

### 仮想ボタン・移動ゾーンの帯内再配置

- [ ] JUMP / SHOT ボタンが下部帯の中に配置され、タップで従来通り入力できる
- [ ] 移動ゾーン／追従パッドが帯のレイアウトと整合し、移動・梯子昇降が従来通り機能する
- [ ] 入力判定の当たり位置と描画されるボタン位置がずれない

### デスクトップでの従来挙動維持

- [ ] タッチ非対応環境では帯が表示されず、フル画面でプレイできる
- [ ] キーボード操作（移動・ジャンプ・ショット・梯子）が従来通り機能する

## 成功指標

- ボス戦で「指でボスが隠れて攻撃が見えない」状況が解消される（定性）
- 帯導入後もプレイ領域でゲームが破綻なく成立する（既存 E2E / 動作確認が通る）

## スコープ外

以下はこのフェーズでは実装しません:

- ボタン配置・サイズ・透明度そのものの大幅なデザイン刷新（帯への移設にとどめる）
- ボスの攻撃パターンや出現位置ロジックの変更
- 弾幕の事前テレグラフ（予兆表示）などの新たな視認補助
- 縦持ち専用UIの新規対応（タッチUIは横持ち前提を踏襲）

## 未決事項

実装フェーズ（add-feature 設計）で判断・確認する:

- 帯の高さの決定値（固定px / 画面高さ比 / ボタン半径基準のいずれか）
- タッチ判定の方式（`game.device.input.touch` / `matchMedia('(pointer: coarse)')` 等）
- 帯を「カメラ viewport 縮小＋zoom補正」で実現するか、論理解像度自体を見直すか
- 帯領域の背景（黒帯 / UI地色 / 装飾）の扱い
