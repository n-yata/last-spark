## 申し送り

### grep一致 ≠ 現役ファイル
調査段階で `raySprite.ts` を「`PreloadScene`/`SpriteRig` から参照されている」という grep 結果だけで
「現役ファイル」と報告しかけたが、実際に `PreloadScene.ts` の該当行を読むと `load.image` 呼び出しは
コメントアウトされており、`raySprite.ts` のヘッダにも「現在は休眠中(将来の再利用のため温存)」と
明記されていた。

**教訓**: ファイルの生死判定は名前の参照有無(grep)だけで判断せず、実際に呼び出し側のコードを読んで
「その参照が生きたパスで実行されるか」まで確認すること。特に「無効化されたが削除せず残している」
実験的機能(feature flag 的なコメントアウト)は grep 一致でも死んでいることがある。

### ドキュメント改訂の優先度判定

という優先度付けが的確だった。今後も複数ドキュメントの一斉改訂が必要な場合、着手前に乖離度の
調査を挟むと、承認を得ながら1ファイルずつ進める際の順序判断がしやすい。
