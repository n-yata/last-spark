# チャージ吸収とボスシールド 設計

## 方針

RAY のチャージを攻撃準備だけでなく、防御と反撃の起点にする。敵の通常弾はチャージ中だけ吸収でき、吸収するとチャージ時間を前倒ししてゲージが伸びる。成功時は小さなスパークと軽いカメラ演出で手応えを出す。

ボスには発光するシールド部位を 1 つ追加し、シールドが残っている間はボス本体 HP を守る。通常弾やビームも少しずつ削れるが、最大チャージ弾だけ大きく削れるため、攻撃タイミングを作る意味が増える。

## 変更点

- `shotControl.ts`
  - チャージ中の蓄積時間を加算する純粋関数を追加する。
- `combatRules.ts`
  - 吸収可能弾の判定、ボスシールドへのダメージ解決を追加する。
- `Player.ts`
  - チャージ中判定と吸収によるチャージ加算 API を追加する。
- `CombatSystem.ts`
  - 敵通常弾がプレイヤーへ当たったとき、チャージ中なら吸収に分岐する。
  - ボスへの命中種別を `Boss.takeDamage` に渡す。
- `Boss.ts`
  - シールド耐久、発光シールド部位、被弾フラッシュを追加する。
- `EffectsManager.ts`
  - 吸収成功の小演出を追加する。
- HUD
  - ボスシールド比率を registry 経由で渡し、ボス HP バーに小さなシールド表示を追加する。

## 検証

- 純粋ロジックのユニットテストを追加する。
- `npm test` と `npm run build` を実行する。
- 可能なら開発サーバーで実機相当の画面確認を行う。
