# 要求内容

## 概要

既存ゲーム LAST SPARK の新規機能追加ではなく、プレイ体験の質(手応え・演出・操作感・難易度)を全体的に引き上げるブラッシュアップ。

## 背景

コア機能(2ステージ+ボス2系統+サウンド+セーブ)は完成済みだが、戦闘のフィードバック演出が欠落しており「敵を倒した爽快感」「被弾の重み」が伝わらない。

- 敵撃破時はただ消えるだけ(`src/entities/Enemy.ts` の destroy のみ)。パーティクル・爆発演出なし
- 画面シェイク・ヒットストップ・被弾フラッシュが皆無
- ヒットエフェクトは単一画像の 180ms tween 1個のみ(`src/scenes/GameScene.ts` の `spawnHitEffect`)
- シーン遷移は全て `scene.start` 直叩きでフェードイン/アウトなし
- タッチ仮想ボタンに押下時の視覚フィードバックなし(`src/ui/TouchControls.ts`)

## 実装対象の機能

### 1. 戦闘演出・手応え強化(最優先)

- 敵撃破時のパーティクル爆発(雑魚は小、ボスは多段・大規模の撃破演出)
- プレイヤー被弾時のカメラシェイク+画面端の赤フラッシュ
- 敵・ボス被弾時の白フラッシュ(点滅)
- チャージ弾命中時・ボス撃破時のヒットストップ(短時間のゲーム停止)
- 演出のチューニング値(シェイク強度・継続時間等)は既存方針に従い `src/config/balance.ts`(または専用 config)に集約
- プレイヤーは「倒した・当てた・食らった」を視覚的に体感できるようになる

### 2. シーン遷移・HUD演出

- 全シーン遷移(Title→Game, Game→Clear/GameOver, Clear→次stage/Title, GameOver→retry/Title)にフェードイン/アウトを追加
- 被ダメ時の HP バー(`src/ui/LifeBar.ts`)の点滅・減少アニメーション
- ボス出現時の BossHpBar(`src/ui/BossHpBar.ts`)出現演出(ゲージが0→満タンに伸びる等)
- シーン切替が瞬間移動でなくなり、被ダメ・ボス出現の状況把握が容易になる

### 3. タッチUIの磨き込み

- 仮想ボタン(`src/ui/TouchControls.ts` / `src/ui/MovePad.ts`)押下時の視覚フィードバック(色変化・スケール等)
- 押下中状態が指で隠れても分かる表現
- タッチプレイヤーが自分の入力状態を確認できるようになる

### 4. 難易度・バランス調整

- stage1→stage2 の難易度カーブが体感できるよう `src/config/balance.ts` の数値を調整(雑魚HP・配置密度・ボス攻撃頻度等)
- 調整はすべて `balance.ts` 内で完結させ、ロジックへのマジックナンバー埋め込みは禁止

## 受け入れ条件

### 戦闘演出・手応え強化

- [ ] 雑魚敵を倒すとパーティクル爆発が表示される(消えるだけではない)
- [ ] ボス撃破時に多段爆発+カメラシェイク+ヒットストップの撃破演出が再生される
- [ ] プレイヤー被弾時にカメラシェイクと画面フラッシュが発生する
- [ ] 敵・ボスに弾が当たると対象が白く点滅する
- [ ] 演出関連のチューニング値が config に集約されている(ロジック内にマジックナンバーがないこと)

### シーン遷移・HUD演出

- [ ] すべてのシーン遷移でフェードイン/アウトが行われる(瞬間切替が残っていない)
- [ ] 被ダメ時に HP バーが視覚的に反応する
- [ ] ボス戦開始時に BossHpBar の出現演出が再生される

### タッチUIの磨き込み

- [ ] タッチ仮想ボタンが押下中に視覚的に変化する

### 難易度・バランス調整

- [ ] stage1→stage2 で難易度の上昇が数値上明確である(`balance.ts` で差分が説明できる)
- [ ] バランス調整値がすべて `balance.ts` 内で完結している

### 全体

- [ ] 既存テスト(21ファイル)がすべてパスする
- [ ] 演出追加後も明らかなフレーム落ちがない(通常プレイ・ボス戦で体感確認)

## 成功指標

- 定性: 敵撃破・被弾・ボス撃破の3場面で「手応えがある」と体感できる
- 定量: 既存テスト全パス、演出値の config 集約率100%

## スコープ外

以下はこのフェーズでは実装しません:

- 新ステージ・新敵種・新ボスの追加
- スコアシステム等の新メカニクス追加
- 新規BGMの追加(既存SE/BGMの流用・軽微な追加SEは可)
- オンライン/ランキング機能

## 未決事項

実装フェーズ(`/add-feature` の設計時)で判断・確認する:

- ヒットストップの長さ・適用範囲(チャージ弾のみか通常弾もか)
- パーティクルの具体的な見た目(Phaser ParticleEmitter のテクスチャ・色)
- 難易度調整の具体的な数値(テストプレイで決定)
