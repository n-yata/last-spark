# 振り返り: boss-story-integration

## 実装完了日

- 2026-07-09

## 計画と実績の差分

- stage4 / stage5 は既存の固有ギミックが十分にあったため、今回は全面改修ではなく `登場 / phase移行 / 撃破余韻` の演出強化を中心に乗せた
- stage3 は新規 `containment` を追加し、要求どおり「収容・拘束」を戦闘の読みへ落とし込めた
- stage6 は phase1 の `雑魚掃討 -> 弱点露出` を追加し、要求どおり「攻撃機会を攻略して作る」構造へ寄せられた
- implementation-validator 相当のサブエージェント検証は、この環境では呼び出し手段が無かったため未実施

## 学んだこと

- 既存のボス個性が揃っている段階では、全ボスを均等に作り直すより「不足している攻略テーマ」だけ差す方が自然に仕上がる
- `Boss.holdFor()` を基底へ追加すると、個別ボス実装を汚さずに演出待機を横断適用できる
- stage 固有ギミックは `GameScene` に生成責務を置き、ボス側は疎結合コンテキスト経由で要求だけ伝える形が安全

## 次回への改善提案

- stage4 / stage5 も、演出だけでなく「撃ち返し機会」や「フェーズごとの盤面変化」をさらに細かく詰める余地がある
- stage6 の弱点露出は視覚で十分伝わるが、将来的には HUD 側にも短い cue を足すと学習がさらに早くなる
- vitest は Windows 環境で worker 終了時に `kill EPERM` 警告を出した。今回はテスト自体は全通過したが、将来 CI/ローカル差分の原因になりうるため追跡候補
