# 要求内容

## 概要

stage3〜stage6 のボス戦を、各ステージの物語転換点と結びついた「攻略して倒す手ごたえ」のある体験へ再設計する。既存の台詞・カットシーンは変更せず、ボス固有ギミックと登場/フェーズ遷移/撃破後の演出を強化して、戦闘の印象を物語の山場として成立させる。

## 背景

現状のプロジェクトは全6ステージ・全6ボスとストーリー実装が揃っている一方で、`docs/story.md` に「ボスとテーマ・物語の接続は未着手」と明記されている。特に stage3〜stage6 は、TERRA 救出、葛藤、決意、最終対決という大きな転換点を担うにもかかわらず、プレイヤー体験としては「最後に強い敵を倒す」以上の意味が薄い。

その結果、物語上は重要な場面でも、プレイヤーの記憶に残るのが台詞だけになりやすい。戦闘そのものに「守る」「侵食される」「見切って反撃する」「弱点を暴く」といった攻略テーマを与え、演出も連動させることで、ストーリーとゲームプレイの接続を強める必要がある。

## 実装対象の機能

### 1. stage3〜stage6 のボスギミック再設計

各ボスに、ステージテーマと結びついた固有の攻略構造を持たせる。

- stage3 は拘束・収容を連想させるギミックで、守る/解放する緊張感を戦闘に持ち込む
- stage4 は汚染領域や浄化装置を読みながら位置取りする戦いにする
- stage5 は高速移動する使者に対し、回避だけでなく反撃機会を見極める戦いにする
- stage6 は複数フェーズを通じて弱点露出や攻撃機会を作る、最終決戦らしい構造にする

### 2. ボス戦演出の強化

各対象ステージのボス戦について、戦闘前後と戦闘中の節目が印象に残るよう演出を追加・強化する。

- ボス登場時に、そのステージの転換点が伝わる導入演出を入れる
- フェーズ遷移時に、攻略状況の変化が視覚的・時間的に明確に分かる演出を入れる
- 撃破後に、既存の物語テキストや遷移へ自然につながる余韻演出を入れる

## 受け入れ条件

### stage3〜stage6 のボスギミック再設計

- [ ] stage3〜stage6 の各ボスに、既存の移動/攻撃パターン強化だけではない「固有の攻略テーマ」が定義され、プレイヤーが対処法の違いを体感できる
- [ ] stage3 のボス戦は、収容・拘束・保護の文脈を感じられるギミックを含み、単なる火力強化版ではない
- [ ] stage4 のボス戦は、汚染や危険領域の変化を読みながら安全地帯と攻撃機会を選ぶ構造を含む
- [ ] stage5 のボス戦は、高速移動や急襲に対して観察し、反撃タイミングを見極める構造を含む
- [ ] stage6 のボス戦は、最終決戦として複数フェーズの役割差が明確で、弱点露出や攻撃機会を攻略して作る構造を含む
- [ ] 各ボスの新規ギミックは当該ボス専用または当該ステージ専用に閉じ込められ、他ステージボスの挙動を意図せず変えない

### ボス戦演出の強化

- [ ] stage3〜stage6 の各ボスに、戦闘開始前の登場演出が追加または強化され、物語上の転換点に入ったことがプレイヤーに伝わる
- [ ] 各ボスのフェーズ遷移時に、攻撃パターンの変化や戦況の切り替わりが分かる専用演出がある
- [ ] 各ボス撃破後に、既存の台詞・カットシーン・クリア遷移へ自然につながる余韻演出がある
- [ ] 新規演出は既存のストーリーテキスト (`src/config/story/`) を改変せずに成立する

### 共通

- [ ] stage3〜stage6 の再設計後も、難易度カーブが破綻せず、後半ステージとしての手ごたえを維持したまま理不尽さを増やしすぎない
- [ ] ボス戦ごとに「何を見て、どう崩すか」がプレイヤーに学習可能で、再挑戦時に上達実感が得られる
- [ ] `npm test` / `npm run typecheck` / `npm run build` が通る

## 成功指標

- stage3〜stage6 の各ボスで、プレイヤーが異なる攻略テーマを明確に体感できる
- 再挑戦時に「見切れた」「対処法が分かった」と感じられる学習価値が増す
- ボス前後の演出が、そのステージの物語上の転換点として自然につながっている

## スコープ外

以下はこのフェーズでは実装しません:

- stage1・stage2 ボスの再設計
- `src/config/story/` の台詞追加・改稿
- 新ステージ、新ボス、新キャラクターの追加
- BGM 全面刷新や大規模な音響仕様変更
- hard 専用の別ギミック追加や裏ボス戦の再設計
