# 要求内容

## 概要

合成BGM(Web Audioで生成する5トラック)を、いまのゲームの雰囲気を保ったまま「もう少し重厚」にする。
土台(低音ドローン)の和音化と、メロディへのハーモニー声部追加により厚み・圧を増す。

## 背景

現状のBGMは `src/config/audio.ts` のデータ定義を `SoundManager` がWeb Audioで合成している。
メロディは単声〜デチューン2声、ドローンは単一semitoneのsine 1声に留まり、薄い。
シャビ(監督)から「いまの雰囲気に沿って、もう少し重厚に」との要望。確認の結果、方針は
「土台を厚く＋声部・ハーモニーも追加(最大限の重厚化)」に決定。

北極星(`docs/story.md`「BGM方針」)の4シーンの方向性は厳守する:

| シーン | 方向性(維持) |
|--------|-------------|
| ステージ探索 | アンビエント・ドローン系。廃墟の静寂と不安感 |
| ボス戦 | 機械的な冷たさ・無機質なビート |
| Stage 3〜(TERRA登場後) | 探索にわずかな「温もり」 |
| エンディング | 静かで余韻を残す人間的な弦・ピアノ系 |

## 実装対象の機能

### 1. ドローンの多声化(土台の和音化)

- `BgmDrone` に追加声(`semitones?: number[]`)を持たせ、ルート＋完全5度＋オクターブ等の
  パッドとして低音の土台を分厚くする。
- 後方互換: 既存の `semitone`/`volume` はそのまま保持。`semitones` 未指定なら従来どおり単声。

### 2. メロディのハーモニー声部

- `BgmTrack` に `harmonies?: number[]`(各ノートへ重ねる半音オフセットの配列)を追加。
- 完全5度上(パワーコード感)やオクターブ下(重み)を重ね、音楽的な厚みを出す。
- 既存の `detuneCents`(2声の揺らぎ)と併用可能にする。

### 3. クリップ管理の一般化(声数→ピーク正規化)

- 声部が増えると合算で音圧が上がりクリップする。声の総数からピーク音量を求める純粋関数
  `voicePeak()` を `soundSynth.ts` に切り出し、`SoundManager` のノート合成で使う。
- 既存の「detune時0.5/単声0.8」という2値固定を、声数ベースの正規化へ置き換える。

### 4. 各トラックのチューニング(雰囲気維持の重厚化)

- 4シーンの方向性を壊さない範囲で、各トラックにドローン和音化／ハーモニー声部を適用する。

## 受け入れ条件

### 重厚化(機能)
- [ ] `boss` がドローン和音化＋ハーモニー(完全5度/オクターブ)で最も重く・無機質になっている
- [ ] `stage` は静寂・不安感を保ったまま、ドローンの和音化で土台だけ厚くなっている(過剰なメロディ重ねをしない)
- [ ] `stageWarm` は長3度(C#5=+4)の温もりを維持したまま厚みが増している
- [ ] `ending` は弦・パッドの厚み(ドローン和音化＋オクターブ下＋デチューン)が増している

### 品質(クリップ・互換)
- [ ] 声部増加後も合成音がクリップしない設計(`voicePeak` でピークを声数正規化)
- [ ] `voicePeak` は声数1で従来相当、声数増で単調減少し、0–1に収まる(純粋関数として単体テスト可能)
- [ ] 既存テスト(`audio.test.ts`/`soundSynth.test.ts`)を壊さない(後方互換)

### 検証
- [ ] `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` がすべて通る
- [ ] 実機(ブラウザ)で各シーンのBGMが鳴り、無音化・破綻していないことを確認

## 成功指標

- 4シーンの方向性(短3度の陰り／長3度の温もり／ボスの無機質／エンディングの弦)が保たれている
- 既存・新規テストが全通過し、ビルドが成功する
- 声部追加によるクリップ(歪み)が発生しない

## スコープ外

以下はこのフェーズでは実装しません:

- 外部音源ファイル(mp3/ogg等)の導入(現行のWeb Audio合成方針を維持)
- 新規BGMトラック・新規SEの追加
- 音量設定UI(optionsMenu)の変更
- ビーム音(BEAM_SOUND)・単発SEの仕様変更
