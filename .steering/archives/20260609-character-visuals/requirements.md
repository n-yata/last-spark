# 要求内容

## 概要

プレイヤー・雑魚敵・ボスの見た目を、手続き生成（Graphics）でパーツ分割した関節構造に
作り直し、歩行・ジャンプ/落下・攻撃・被弾のモーションを与えて、「ただの丸」から
生命感のあるキャラクターへ引き上げる。外部素材は追加しない。

## 背景

- 現状、プレイヤー・雑魚敵・ボスはすべて Phaser スプライトだが、見た目は
  `src/scenes/PreloadScene.ts` の `makeCharacter()` が生成するプレースホルダ
  （角丸の箱＋中央の発光コア）に過ぎない。
- `anims`（アニメーション定義）がプロジェクト内に一つも存在せず、移動・ジャンプ・
  発射・被弾のいずれも絵としては静止画のまま。動いているのは座標移動・`flipX`・
  被弾点滅/ティントだけで、キャラクターとしての生命感がない。
- MVP は一貫して「コードでプレースホルダ生成」方針で、外部アセットを持たない。
  素材調達のボトルネックを避けつつ品質を上げるため、手続き生成を強化する。

### 確定した方針（合意済み）

- **作り方: 手続き生成の強化**（外部スプライト不使用、依存追加なし、コードで完結）
  - 頭・胴・腕・脚などをパーツ単位で扱い、Phaser の `Container` で関節として組み立てる。
  - アニメはスプライトシートではなく**トゥイーン/手続き駆動**で表現する。
  - 暗め基調＋ネオン発光の既存 world 観を継承する。
  - 将来「外部スプライトシート + `anims`」へ移行できるよう、テクスチャキーの集約
    （`src/config/assetKeys.ts`）と基底クラスの抽象を保つ。
- **キャラ像**
  - プレイヤー =「最後のロボット」。**人型ロボット**。プロポーションは
    **スタンダード・中間型**（細身と重厚の中間、頭身バランスの取れた標準体型）。
    **丸みのあるヘルメット型の頭**（発光する目/バイザー）、標準的な胴と両脚、
    **片腕がアームキャノン**。ラン&ガンアクションの王道テイストを取り入れるが、
    **造形・配色・名称は本作オリジナル**（世界観のネオン発光色）で作る。
  - 敵/ボスは設定通りメカ寄り: walker=二足/多脚の歩行警備ロボ、turret=脚のない
    固定砲台（砲身が向く）、ボス=重量感のある大型警備機。プレイヤーと色/シルエットで
    明確に区別する。
- **IP制約（必須）**: ロックマン等の既存作品の固有意匠（顔・配色・ヘルメット形状等）の
  模倣や、その名称・固有名のコード/アセット/ドキュメントへの使用は禁止
  （`docs/architecture.md` / `docs/functional-design.md` の方針に従う）。

## 実装対象の機能

### 1. キャラのパーツ化・シルエット刷新

- 頭・胴・腕・脚などのパーツで構成された、ロボSF風のシルエット（丸ではない）。
- プレイヤーはスタンダード体型の人型ロボット（丸みのあるヘルメット型の頭 /
  標準的な胴・両脚 / 片腕アームキャノン）。ラン&ガンの王道テイストを
  本作オリジナルの造形・ネオン配色で表現する。
- 敵/ボスはメカ寄り（walker=歩行警備ロボ、turret=固定砲台、ボス=大型警備機）。
- プレイヤー / walker / turret / ボスを、サイズ・配色・パーツ構成で作り分ける。

### 2. 歩行モーション

- 横移動中に脚（と腕）が周期的に動き、停止すると静止に戻る。
- 進行方向に応じて左右の向きに追従する。

### 3. ジャンプ/落下の姿勢

- 上昇・下降・接地で姿勢が変化する（例: 踏み切りで縮む→空中で伸びる→着地で潰れる）。

### 4. 攻撃モーション

- 発射（プレイヤー / turret / ボス）に合わせた予備動作・反動。
- プレイヤーのチャージ時の溜め表現。

### 5. 被弾・やられ表現

- 被弾でのけぞり/フラッシュ、無敵中の点滅（既存挙動の置換）、撃破時のやられ演出。
- ボスの stagger（のけぞり）状態を姿勢として表現する。

## 受け入れ条件

### キャラのパーツ化・シルエット刷新

- [ ] プレイヤー・walker・turret・ボスが、丸/箱ではなくパーツ構成のシルエットで表示される。
- [ ] プレイヤーがスタンダード体型の人型ロボット（ヘルメット型の頭・胴・両脚・片腕アームキャノン）として表示される。
- [ ] 4 種が配色・サイズ・形状で互いに区別できる。
- [ ] ロックマン等の固有意匠の模倣・名称使用がない（造形/配色/命名がオリジナル）。

### 歩行モーション

- [ ] 横移動中に脚（または相当パーツ）が動き、静止すると動きが止まる。
- [ ] 進行方向に応じて左右の向きが反転する（既存 flipX 相当を維持）。

### ジャンプ/落下の姿勢

- [ ] ジャンプの上昇・下降・着地で姿勢が視覚的に変化する。

### 攻撃モーション

- [ ] プレイヤーの通常/チャージ発射時に予備動作または反動が見える。
- [ ] turret・ボスの発射時にも攻撃モーションが見える。

### 被弾・やられ表現

- [ ] 被弾時にのけぞりまたはフラッシュ等のフィードバックがある。
- [ ] 無敵中の点滅が従来同様に機能する（リグレッションなし）。
- [ ] ボスの stagger 状態が姿勢/見た目で判別できる。

### 非機能

- [ ] 既存の移動・当たり判定・発射・HP/フェーズ等のゲームロジックが壊れない。
- [ ] 物理ボディサイズ（`body.setSize`）はゲームバランス維持のため従来値を踏襲する。
- [ ] 既存ユニットテストが通り、必要なら表現用のテストを追加する。
- [ ] 外部アセット（画像/フォント等）や新規 npm 依存を追加しない。

## 成功指標

- 定性: 起動して動かしたとき、4 キャラが「丸」ではなく生きて動くキャラに見える。
- 定性: 歩行・ジャンプ・攻撃・被弾の状態が、見た目だけで判別できる。
- 定量: 既存テストが全て green。表示まわりのフレーム性能が体感で劣化しない。

## スコープ外

以下はこのフェーズでは実装しません:

- 外部スプライトシート/ドット絵素材の用意と `anims` 移行（将来案として保留）。
- 新キャラ・新ステージ・新規攻撃パターン等のゲーム内容追加。
- 弾・地形・UI ビジュアルの刷新（今回はキャラ本体に集中）。
- サウンドの追加・変更。
- ロックマン等の既存IPの固有デザイン/配色/名称の模倣（方針・法務リスクのため恒久的に対象外）。
