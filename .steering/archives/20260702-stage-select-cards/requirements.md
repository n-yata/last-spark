# 要求内容

## 概要

タイトル画面のステージセレクトを、テキストの縦並びリストから「カード式」のUIへ格上げする。各ステージカードに背景テーマのミニプレビュー・クリア状況・ベストタイム・ロック状態を表示し、進捗が一目で見える画面にする。

## 背景

現状のステージセレクトは monospace テキストボタンが縦に並ぶだけで、6ステージ+周回要素(New Game+)が揃った現在のコンテンツ量に対して情報量・魅力が不足している。プレイヤーが毎回通る画面であり、UI強化の投資対効果が最も高い。セカンダリーペルソナ(世界観・ビジュアル重視層)への訴求にも直結する。

## 実装対象の機能

### 1. カード式ステージ一覧(グリッド配置)
- 6ステージをカードのグリッド(横3×縦2を基本、画面サイズから算出)で表示する
- 各カードに「STAGE n」番号とステージ名を表示する
- カードをタップすると該当ステージで本編を開始する(従来と同じ導線)

### 2. 背景テーマのミニプレビュー
- カード上部に、そのステージの背景テーマ(`stageBackground.ts` の skyTop/skyBottom/accent/layers)から手続き生成したミニプレビューを描く
- 空グラデーション(lerpColor による帯)+ シルエット列(`generateSilhouetteColumns`、決定論)+ アクセント灯で構成する
- 外部アセットは追加しない(既存の手続き生成方針を踏襲)

### 3. 進捗表示(クリア済みマーク・ベストタイム)
- 今周回でクリア済みのステージに「CLEAR」バッジを表示する
- ベストタイム(`bestTimeMs`、周回をまたいで保持)があれば `BEST m:ss.d` 形式で表示する

### 4. ロック表示(段階解放)
- ステージの解放条件: stage1 は常に解放。stage n は「stage n-1 を一度でもクリアしたことがある(今周回の `clearedStages` または過去記録 `bestTimeMs`)」場合に解放
- **過去記録(`bestTimeMs`)を解放判定に含めることで、周回(New Game+)で `clearedStages` がリセットされても一度到達したステージは選び直せる**(既存の「任意のステージから始められる」体験を周回プレイヤーから奪わない)
- 未解放カードは暗転+「LOCKED」表示にし、タップしても開始しない

## 受け入れ条件

### カード式ステージ一覧
- [ ] 6ステージがカードのグリッドで表示される(画面内に収まり、はみ出さない)
- [ ] 解放済みカードのタップで該当ステージが開始される(効果音・遷移は従来どおり)
- [ ] BACK でオーバーレイが閉じ、タイトルのスタート導線が復帰する
- [ ] 高DPI規約(`scaled()`/`scaledFontPx()`)に従い、モバイル実機相当の解像度でレイアウトが崩れない

### ミニプレビュー
- [ ] 6ステージそれぞれのカードに、テーマ色(空・シルエット・アクセント)を反映した異なるプレビューが描かれる
- [ ] プレビューは決定論的(再描画・リサイズで同一の見た目)

### 進捗表示
- [ ] 今周回クリア済みのステージに CLEAR バッジが出る
- [ ] ベストタイムを持つステージに `BEST m:ss.d` が表示される
- [ ] 進捗なし(初回起動)ではバッジ・タイムが表示されない

### ロック表示
- [ ] 初回起動時は stage1 のみ解放、stage2〜6 は LOCKED 表示
- [ ] stage n をクリアすると stage n+1 が解放される
- [ ] 周回リセット後(`clearedStages` 空)でも、過去にクリアした記録(`bestTimeMs`)があるステージまでは解放されたまま
- [ ] LOCKED カードをタップしてもステージが開始されない

## 成功指標

- ステージセレクトを開いた瞬間に「どこまで進んだか・次はどこか」が読み取れる
- 既存機能(ステージ開始・BACK・タイトルとの排他)の回帰がない(既存テストが全て通る)

## スコープ外

以下はこのフェーズでは実装しません:

- リザルトランク(S/A/B)の表示(別作業。カードにタイム表示の枠だけ用意)
- カード内のステージスクリーンショット/サムネイル画像(手続き生成プレビューで代替)
- オプションメニュー・ポーズ画面のステージ移動パネルのカード化(従来のテキストリストを維持)
- 難易度・周回数のカード内表示
