## 実装後の振り返り

### 実装完了日
2026-06-14

### 計画と実績の差分

**計画と異なった点**:
- カットシーンSVGの明度改善手法を、設計の未決事項のとおり実装時に確定。当初案の「ビネット不透明度を下げ + 主要色を個別に持ち上げる」(多数の色を手編集)ではなく、**各SVGに sRGB の feComponentTransfer(線形変換 slope/intercept)フィルタを1つ追加し、全描画を `<g filter="url(#lift)">` で包む**方式を採用。理由: (1) 1枚あたり2箇所の編集で済み破綻リスクが低い、(2) 作者の相対トーンを保ったまま暗部を一括で持ち上げられ、絵を壊さない、(3) ステージごとにゲイン/黒持ち上げを1パラメータで調整できる。

**新たに必要になったタスク**:
- レビュー指摘対応: stage6 の skyTop と layers[0] が同輝度(lum18)でモノリス頭頂のコントラストが0になる問題を、skyTop を #1a1928(lum23)へ引き上げて解消(最暗トーンは維持)。

**技術的理由でスキップしたタスク**: なし(全タスク完了)。

### 学んだこと

**技術的な学び**:
- Phaser の `load.svg` はブラウザでSVGをラスタライズするため、`defs` 参照(グラデーション)だけでなく **SVGフィルタ(feComponentTransfer 等)も実描画に適用される**。実機(stage5カットシーン)でフィルタ適用を確認済み。「暗いSVGアセットを絵を壊さず一括で明るくしたい」場合の再利用パターンになる。
- 実機検証で `window.lastSpark`(Phaserゲーム実体)から `scene.start('GameScene', { stageId })` で任意ステージを直接起動でき、Playwright の要素スクショで実フレームを取得できる。WebGL canvas は `preserveDrawingBuffer=false` のため `canvas.toDataURL()` は黒になる→Playwrightのスクショ機能を使うこと。
- stage1 が「ちょうどいい」のは空がシルエットより十分明るくコントラストが付いているため。暗いステージは「空とシルエットの双方が暗部に沈み無コントラスト」だった。明度改善は**空を中心に上げ、シルエットは空より暗いが暗黒でない水準**にするのが要点。

**プロセス上の改善点**:
- 色という定性的な対象でも、実機スクショ + 画素輝度の実測(空上端R値の前後比較)で「効いているか」を定量確認でき、フィルタ未適用の取りこぼしを防げた。

### 次回への改善提案
- 背景トーンの定性評価は最終的にシャビの目視が必要。実機スクショを用意してレビューに添えると合否判断が速い。
- 暗部視認性の調整は「skyTop(空上端)と layers[0](最奥シルエット)の輝度差」を最低5程度確保するのを目安にすると、頭頂が空に被ってもシルエットが読める。
