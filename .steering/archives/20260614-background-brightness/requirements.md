# 要求内容

## 概要

プレイ中のステージ背景(stage2〜6)と、ストーリー演出(カットシーン)の背景画像が暗すぎて視認しづらい問題を解消する。各ステージ・各シーンの世界観(色調・雰囲気)は維持したまま、stage1相当の視認性まで明度を引き上げる。

## 背景

- ステージ中の背景は stage1 だけが「ちょうどいい明るさ」で、stage2〜6 は暗すぎて何が描かれているか判別しづらい。
  - 原因: 背景の空グラデーション色とシルエット色を定義する `src/config/stageBackground.ts` の各テーマ(`skyTop`/`skyBottom`/`layers[].color`)が、stage1 に比べて stage2〜6 で明度が大きく低い。加えて stage4/5/6 はカメラ背景色(`src/config/stage1.ts` の `StageData.backgroundColor`)も別途暗く設定されている。
- 途中のストーリー背景(カットシーン)も暗すぎて、何が描かれているか判別しづらい。
  - 原因: `CutsceneScene` が敷く静止画 `public/assets/cutscenes/*.svg`(全5枚)の絵全体が暗い。
- これらは「世界観の演出」と「視認性」のバランスが視認性側に振り切れておらず、プレイ体験・物語没入を損なっている。本フェーズで視認性を回復する。

## 実装対象の機能

### 1. プレイ中ステージ背景(stage2〜6)の明度改善

- `src/config/stageBackground.ts` の stage2〜6 テーマについて、空グラデーション(`skyTop`/`skyBottom`)とシルエットレイヤー色(`layers[].color`)の明度を引き上げ、stage1 相当の視認性にする。
- 各ステージの色相・世界観(stage2=冷たいシアンの立坑 / stage3=人工照明の収容施設 / stage4=毒性の黄緑の汚染地帯 / stage5=青い金属の外縁部 / stage6=不吉な赤の支配中枢)は維持し、一律な明るさにはせず**ステージごとに個別調整**する。
- stage6(支配中枢/ラスボス)は意図的に漆黒だが、本改善の対象に含める。**影の核としての不吉さ・威圧感(暗いトーン・赤の脈動)は残しつつ**、プレイヤーが地形・シルエットを判別できる最低限の明度まで引き上げる。
- stage4/5/6 のカメラ背景色(`StageData.backgroundColor`)も、背景テーマの明度引き上げと矛盾しないよう必要に応じて併せて調整する(地平線下・背景の保険色として暗すぎて浮かないこと)。
- アクセント発光色(`accent`: 窓灯/監視灯/コア光)は世界観の識別要素なので、必要なら視認性向上に資する範囲で微調整に留める(色相は変えない)。

### 2. カットシーン背景画像(SVG 5枚)の明度改善

- `public/assets/cutscenes/` の以下5枚のSVGを直接編集し、絵全体を明るくして何が描かれているか判別できるようにする。
  - `stage1-intro.svg`
  - `stage3-rescue.svg`
  - `stage4-intro.svg`
  - `stage5-intro.svg`
  - `stage6-ending.svg`
- 各カットシーンの場面・雰囲気(色調・構図・キャラシルエットの意図)は維持し、暗部の沈み込みを持ち上げて視認性を回復する。
- 既存の本文テキスト表示(`CutsceneScene` が画面上部に出す字幕)の可読性を損なわないこと。明るくした結果、字幕と背景の明部がぶつかって読みにくくならないようにする(必要なら字幕側の縁取り/影は既存実装の範囲で確認)。

## 受け入れ条件

### プレイ中ステージ背景(stage2〜6)の明度改善

- [ ] stage2〜6 をプレイ(または開発モードのステージ選択)で実際に表示し、空・シルエット・地形が stage1 と同等に判別できる。
- [ ] 各ステージの色相・世界観が維持されている(stage2=シアン系 / stage3=白青系 / stage4=黄緑系 / stage5=青系 / stage6=赤を差し色にした暗藍系)。一律の同じ明るさになっていない。
- [ ] stage6 は地形・シルエットが判別できる一方で、影の核としての不吉さ・威圧感(暗めのトーン)が残っている。
- [ ] stage4/5/6 でカメラ背景色と背景テーマの明度に不自然な段差(地平線下だけ極端に暗い等)が生じていない。
- [ ] 既存のステージ背景テスト(`tests/unit/config/stageBackground.test.ts`)が通る。色値変更でテストの期待値修正が必要な場合は、テスト意図を保ったまま更新する(視認性検証を骨抜きにするハードコードは禁止)。
- [ ] `npm run lint` / typecheck / `npm test` / build が通る。

### カットシーン背景画像(SVG 5枚)の明度改善

- [ ] 対象5枚のSVGを表示(該当カットシーン再生)し、描かれている場面・モチーフが判別できる明るさになっている。
- [ ] 各カットシーンの場面・雰囲気が維持されている(別物の絵になっていない)。
- [ ] カットシーン本文の字幕が、明るくした背景の上でも読める。
- [ ] SVGとして正しくロード・描画される(`load.svg` 失敗やレンダリング崩れがない)。

## 成功指標

- 定性: stage1 を基準にしたとき、stage2〜6 のプレイ背景および5枚のカットシーン背景が「暗すぎて何が何だか分からない」状態から「世界観を保ったまま視認できる」状態へ改善している(シャビの目視確認で合格)。
- 定性: 各ステージ/各シーンの個性(色調・雰囲気)が、明度改善後も識別できる。

## スコープ外

以下はこのフェーズでは実装しません:

- 背景の構図・モチーフ・シルエット形状そのものの作り直し(明度/色の調整に限定。描画ロジック `backgroundPainter.ts` の構造変更や新規アセット追加は行わない)。
- stage1 のプレイ背景の変更(現状が適正なため触らない)。
- カットシーン背景を新規に描き起こす/別アセットへ差し替えること(既存SVGの色編集に限定)。
- 明度をユーザーが調整できる設定(輝度オプション等)の追加。
- StoryOverlay(プレイ中の浮遊テキスト)の見た目変更。

## 未決事項

実装フェーズ(`/add-feature`)で判断・確認する点:

- 明度引き上げの具体的な目標値(各 `skyTop`/`skyBottom`/`layers[].color`/`backgroundColor` の新しい hex)。stage1 の明度を参照点に、ステージごとに設計フェーズで決める。
- カットシーンSVGの明度改善の具体手法(各要素の塗り色を直接持ち上げる / 全体に明色のオーバーレイ矩形を重ねる / フィルタ調整 等)。素材の作り(ベタ塗りかグラデーションか)を実装時に確認して選ぶ。
- `tests/unit/config/stageBackground.test.ts` が色値をどこまで固定的に検証しているか実装時に確認し、必要な期待値更新の範囲を確定する。
