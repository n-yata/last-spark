# 要求内容

## 概要

`docs/story.md`「BGM方針」に基づき、シーン別の BGM を拡張する。現行の BGM 3トラック（title/stage/boss）を、story.md の4方針（探索／ボス／TERRA登場後の温もり／エンディング）に沿って整える。

## 背景

現行 `SoundManager` は外部音源を持たず、`config/audio.ts` の合成仕様から Web Audio で手続き生成する方式（知財方針準拠）。story.md ではビジュアルトーンと同じ方向性で4シーンの BGM 方針を確定済みだが、実装には反映されていない。このブロックは他ブロックと独立して並行実装できる（ステージ実装と依存しない）。

## 実装対象の機能

### 1. 探索 BGM（アンビエント・ドローン系）
- 廃墟の静寂と不安感。「音の風景」に近い合成

### 2. ボス BGM（無機質な電子音楽）
- ECLIPSEの機械的な冷たさを反映したビート（現行 boss トラックの方針整合・必要なら調整）

### 3. TERRA登場後の温もり（Stage 3以降）
- 探索 BGM にわずかに温もりを混ぜたバリエーション
- Stage 3 クリア以降（TERRA同行後）に適用される切替

### 4. エンディング BGM
- 静かで余韻を残す、人間的な楽器音（弦・ピアノ系の合成）

## 受け入れ条件

- [ ] 探索／ボス／温もり／エンディングの4方針が合成仕様として実装される
- [ ] TERRA同行後（Stage 3以降）に探索 BGM が温もりバリエーションへ切り替わる
- [ ] エンディングシーンで専用 BGM が再生される
- [ ] ミュート・音量設定（GameSettings）が全トラックに反映される
- [ ] `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` が通る

## 成功指標
- BGM がビジュアルトーン（暗い廃墟＋発光アクセント）と一貫したサウンド体験を提供する
- TERRA登場前後で音の温度差を感じられる

## スコープ外
- 外部音源ファイルの導入（合成方式を維持）
- 効果音（SE）の追加・変更
