# 要求定義: サウンド演出（BGM/SE）の追加

## 概要

Web Audio API でプログラム生成した BGM / 効果音(SE) を導入し、ゲーム全体に音の演出を加える。外部音源ファイルは使わず、合成音のみで完結させる。設定UI（ミュート/音量調整）は今回のスコープ外とし、`GameSettings` の既定値で再生する。

## 背景

- 現状、サウンドは未実装（`public/assets/audio/` は予約のみ・空、再生コードなし）。
- 永続化層 `SaveManager` に `GameSettings { muted, bgmVolume, seVolume }` が定義済みだが未活用。
- テクスチャを Graphics で生成し完全オフラインを貫く方針があり、音も同様にプログラム生成が自然。
- 世界観「退廃の中の希望／ネオン(シアン #37f7d8・ピンク #ff2d55)」にレトロ電子音が合致。

## 実装対象の機能

### 1. サウンド再生基盤（Web Audio）
- `AudioContext` を用いた合成音再生の仕組みを 1 つの System として用意（例: `src/systems/SoundSystem.ts`）。
- SE は短い合成音（矩形波/ノコギリ波/ノイズ + エンベロープ）、BGM は合成音のループとして生成。
- `GameSettings` の `bgmVolume` / `seVolume` / `muted` を読み取り音量に反映する（UI は作らないが値は尊重）。

### 2. モバイル autoplay 制約への対応
- 最初のユーザー操作（タイトルのタップ等）で `AudioContext` を resume し、音が出ない問題を回避する。

### 3. BGM（シーン連動）
- タイトル / 通常ステージ / ボス戦 で BGM を切り替える。シーン遷移時に適切に停止・開始する。

### 4. 効果音(SE)（イベント連動）
既存のコアループ/UIイベントにフックする:
- ジャンプ、チャージ開始・チャージ完了、通常弾発射、チャージ弾発射、敵被弾、ボス被弾、プレイヤー被弾、敵撃破、ボス撃破、ステージクリア、ゲームオーバー、UIボタンタップ。

## 受け入れ条件

- [ ] 外部音源ファイル（mp3/ogg/wav 等）を一切追加せず、Web Audio API の合成のみで音が鳴る。
- [ ] タイトル / 通常ステージ / ボス戦 でそれぞれ異なる BGM がループ再生され、シーン遷移時に前 BGM が停止し次 BGM に切り替わる（多重再生しない）。
- [ ] 上記「効果音」13 種のイベントで、対応する SE が発火する。
- [ ] モバイル/デスクトップとも、最初のタップ以降は音が確実に鳴る（autoplay ブロックで無音にならない）。
- [ ] `GameSettings.muted = true` を与えると無音、`bgmVolume`/`seVolume` を変えると相対音量が変わる（コードレベルで設定値を尊重。UI からの操作は対象外）。
- [ ] `AudioContext` 非対応/生成失敗時も例外で進行不能にならず、無音でゲームが継続する（SaveManager の堅牢方針に倣う）。
- [ ] 既存のテスト/ビルドが通る（`npm run build` / 既存 Vitest）。

## 成功指標

- 定性: プレイ時に「ジャンプ・射撃・撃破・ボス戦」で音のフィードバックが得られ、手触りが向上したと体感できる。
- 定量: 追加音源ファイル数 = 0（完全オフライン維持）。主要 13 イベントの SE カバレッジ 100%。

## スコープ外

- ミュートボタン / 音量スライダー等の **設定UI**（今回作らない）。
- 外部音源ファイルの導入・差し替え（フリー素材・自前音源）。
- 高度な DSP（リバーブ/コンプレッサ等の本格的エフェクトチェーン）。
- BGM のインタラクティブ変化（フェーズ連動のレイヤー切替など）。

## 未決事項

実装フェーズ（`/add-feature`）で判断・確認する:
- SoundSystem を Phaser の SoundManager 経由にするか、`AudioContext` を直接保持する独立 System にするか（Web Audio 合成主体なので後者が有力）。
- 各 SE/BGM の具体的な音色パラメータ（波形・周波数・エンベロープ・テンポ）。
- BGM ループの生成方式（短いフレーズのスケジュール再生 vs ノードのループ）。
- SE トリガーを各 Entity/System から発火させる際の結合方法（イベントバス/registry/直接呼び出し）。
