# 要求内容

## 概要

ゲーム「LAST SPARK」のストーリーを、テーマ **「心とは、選ぶことだ。」** を背骨に全面リライトする。
本フェーズは「脚本サブエージェント競作」に渡す **企画ブリーフ（STEP1-5の設計）** を確定・固定することが目的。
実テキスト（台詞）の作成・推敲は後続フェーズ（サブエージェント競作）で行う。

## 背景

現行ストーリー（`src/config/story/` が正本）には、以下の構造的な不自然さがある（壁打ちで特定済み）:

1. **三重の繰り返し**: 「導入カットシーン → ステージ開始テキスト → 開始時の内心」がほぼ同内容を繰り返す
   （特に Stage1・Stage4 が顕著）。物語が前進せず単調になる最大要因。
2. **感情・関係性の進行が速すぎる**: テラとの絆が1ステージで完成。レイの揺らぎが早く根拠が薄い。
3. **伏線が宙に浮く**: Stage2「引き寄せられる」が回収されない。Stage6の覚醒「私と同じ何か」が唐突。
4. **「転（どんでん返し）」が不在**: 一本道で予定調和。山場のひっくり返りがない。
5. その他: あの声(管理AI)の脅し文句が単調・反復、守る対象が「人類全体／テラ個人」で揺れる、
   テラのStage6離脱・再合流が描かれない、Stage1導入テキストの誤字混入（`here ここに`）。

設定そのものは自由に書き換えてよい（シャビ判断）。名前「ECLIPSE」は不採用。

## 実装対象の機能

### 1. 企画ブリーフの確定（本フェーズ＝STEP1-5）
- テーマ／ログライン／キャラのアーク／敵＝鏡／世界観方針／プロット骨子（起承転結＋転）を `design.md` に確定。
- 脚本サブエージェントが迷わず競作できる「指示書」として成立させる。

### 2. 脚本チーム競作（後続＝STEP6-7）
- 複数の脚本エージェントに、異なる切り口で「箱書き＋4つの口の実テキスト」を競作させる。
- 編集長エージェントが採点・講評し、勝ち案へ他案の長所を接ぎ木して統合。

### 3. 批評・推敲・統合（後続＝STEP8）
- 感情曲線・伏線・整合性・制約遵守（描画制約／書き方の原則）を批評エージェントが点検。
- 穴を潰して完成稿にする。

### 4. 正本への反映（後続）
- `docs/story.md`（北極星）を完成稿で正式更新。
- `src/config/story/`（stage1-6 + cutscenes）の実テキストを差し替え、`docs` と同期。

## 受け入れ条件

### 企画ブリーフ（本フェーズ）
- [ ] テーマが一文で確定している（「心とは、選ぶことだ。」）
- [ ] ログライン・主要キャラ（レイ／テラ／管理AI）のアークが明記されている
- [ ] 「転」が確定し、6ステージ×起承転結に割り当てられている
- [ ] 伏線→回収の対応が一本の線で追える
- [ ] ゲームの器・描画・書き方の制約が「動かさない前提」として明記されている
- [ ] シャビの承認を得ている

### 後続フェーズ（参考・本フェーズのスコープ外）
- [ ] 4つの口（ステージ開始／あの声／レイ内心／テラ台詞）の実テキストが全6ステージ分そろう
- [ ] 三重の繰り返しが解消され、各テキストが役割分担されている
- [ ] `docs/story.md` と `src/config/story/` が一致している

## 成功指標

- プレイ後に「テーマが伝わり、感動が残る」物語になっていること（定性）。
- 旧版で指摘された5つの不自然さ（繰り返し／進行速度／伏線／転／細部）がすべて解消されていること。

## スコープ外

本フェーズ（企画ブリーフ確定）では以下は実施しない:

- 実テキスト（台詞）の作成 → 後続の脚本チーム競作で行う
- `src/config/story/` のコード変更 → 完成稿確定後
- ボスの行動パターン・ゲームバランスの変更（物語と接続する方針は設計に含むが、実装は別）
- BGM・ビジュアルの変更

## 参照ドキュメント

- `docs/story.md` - ストーリー設定書（Story Bible・北極星。本リライトの対象）
- `src/config/story/` - 現行ストーリーテキストの正本（stage1-6 / cutscenes / index）
- `src/types/story.ts` - テキスト種別・表示の型定義（4つの口の制約）
