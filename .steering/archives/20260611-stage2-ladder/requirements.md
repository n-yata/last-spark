# 要求内容

## 概要

新ステージ stage2 を追加し、梯子(ladder)で上下移動するギミックと、浮いてる足場を
下から通り抜けて上に乗れる「すり抜け床(ワンウェイ床)」を導入する。プレイヤーは移動パッドの
上下操作で梯子を登り降りし、すり抜け床と組み合わせた縦のある地形を攻略して、ステージ終端の
ボスを撃破するとクリアになる。

## 背景

現状は stage1 のみで、遊びが横移動＋ジャンプに限られる。ステージバリエーションとリプレイ性を
高めるため、続きのステージと「縦の移動」という新要素を加えたい。また現状の浮遊足場は両面に
当たり判定があり下から登れないため、梯子と縦移動を活かすにはすり抜け床への変更が必要になる。
既存は複数ステージ・拡張前提の設計（`getStageData()` / `StageData` 型）だが、上下入力・梯子・
すり抜け床は未実装のため新規に追加する。

## 実装対象の機能

### 1. すり抜け床（ワンウェイ床）
- 浮いてる足場(platform)を、下から通り抜けて上に乗れる当たり判定に変更する
- 上からは従来どおり着地でき、下からジャンプしても頭をぶつけず通過できる
- 地面(ground)は対象外で、従来どおり壁/床として機能する
- この仕様は stage1・stage2 の浮遊足場すべてに共通適用する

### 2. 梯子ギミック（上下移動）
- ステージデータに梯子領域（`ladders` など）を定義できるようにする
- プレイヤーが梯子に重なっている間、移動パッドの上下入力で登り降りできる
- 梯子につかまっている間は重力の影響を受けず、上下入力で垂直速度を制御する
- 梯子を登りきると、その上にある足場（すり抜け床）に乗れる動線にする
- 梯子の見た目（テクスチャ/描画）と、登り中のキャラクター表現を用意する

### 3. 新ステージ stage2
- stage2 の `StageData`（地形・梯子・敵配置・ボス出現）を新規定義する
- 梯子とすり抜け床を使って到達する高所/経路を含む、縦の攻略要素を持つ地形にする
- ステージ終端で既存のボストリガー機構によりボス戦に突入し、撃破でクリア
- stage1 クリア後に stage2 へ進む流れを新設する

## 受け入れ条件

### すり抜け床
- [ ] 浮いてる足場の下からジャンプすると、頭をぶつけずに通過して上に乗れる
- [ ] 足場の上からは従来どおり着地でき、足場をすり抜けて落ちたりしない
- [ ] 地面(ground)は従来どおり壁/床として機能し、下から抜けられない
- [ ] stage1 の既存足場もすり抜け仕様になっている（下から登れる）

### 梯子ギミック
- [ ] ステージデータに梯子領域を定義でき、ゲーム画面に梯子が表示される
- [ ] プレイヤーが梯子に重なって上入力すると上方向へ、下入力すると下方向へ移動する
- [ ] 梯子につかまっている間は落下（重力）せず、入力を離すとその位置で静止する
- [ ] 梯子を登りきると上の足場に乗れる（てっぺんで詰まったり弾かれたりしない）
- [ ] 梯子から外れる（最上部/最下部や横移動で離れる）と通常の重力挙動に戻る
- [ ] 梯子上での移動中、登り中であることが分かる表示（アニメ/姿勢）になる
- [ ] 既存の左右移動・ジャンプ・ショット操作が梯子追加後も従来どおり機能する

### 新ステージ stage2
- [ ] stage1 をクリアすると stage2 が開始される
- [ ] stage2 に梯子で登る/降りる縦の攻略箇所が少なくとも1つ存在する
- [ ] stage2 の地形・敵が設計どおり配置され、奈落落下死など既存ルールが機能する
- [ ] stage2 終端でボス戦に突入し、ボスを撃破するとクリア演出に進む
- [ ] stage1 は単独でプレイ・クリアでき、既存挙動（すり抜け床化を除く）を壊さない

## 成功指標

- stage2 を最後までプレイでき、梯子を使う箇所を通過しないとクリアできない動線になっている
- 梯子の操作（パッド上下）が説明なしで直感的に登り降りできる
- すり抜け床により「下から足場に乗る」操作が自然にでき、頭ぶつけのストレスが解消されている
- 既存 stage1 がリグレッションなく通しでクリアできる

## スコープ外

以下はこのフェーズでは実装しません:

- ステージセレクト画面（タイトルから任意ステージを選ぶ機能）
- stage3 以降の追加、ステージ進行のセーブ/再開（ステージ単位のコンティニュー）
- 梯子以外の新ギミック（ロープ、リフト、ワープ等）
- 梯子用の専用ボタンUI（上下操作は移動パッドのY成分で行う）
- 地面(ground)のすり抜け化（対象は浮遊足場のみ）
- すり抜け床を下入力で能動的にすり抜けて降りる操作（下りはジャンプ/端から、で最小限）

## 未決事項

実装フェーズ（`/add-feature` の設計フェーズ）で判断・確認する:

- stage1→stage2 の遷移方法（`ClearScene` を経由して続行か、専用の中継演出か、直接遷移か）
- stage2 クリア後の扱い（最終クリア演出/エンディングか、タイトルへ戻すか）
- 現在ステージIDの保持先（GameScene 引数 / registry / SaveManager のいずれか）
- すり抜け床の実装方式（`body.checkCollision.down` 制御 / collider の processCallback で
  上から来た時だけ衝突、など）と、敵・ボスにもすり抜けを適用するか（プレイヤー限定が無難）
- 梯子につかまる条件の厳密さ（重なり＋上下入力で把持、横移動/ジャンプで離脱 等）
- パッドY成分の判定としきい値（誤反応防止のため梯子重なり時のみ縦入力を有効化）
- 梯子上でのジャンプ・ショットの可否
