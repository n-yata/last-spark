# 要求内容

## 概要

ゲームに統合的な「オプションメニュー」を導入する。タイトル画面とゲームプレイ中（ポーズ）の双方から開け、
音量調整・操作説明の確認・ステージ移動（選択/タイトルへ戻る/リトライ）を一箇所で行えるようにする。

## 背景

現状、設定の永続化基盤（`GameSettings`: muted/bgmVolume/seVolume）と `SaveManager.updateSettings()`、
`SoundManager.applySettings()` は実装済みだが、**プレイヤーがそれを操作するUIが存在しない**。
また、プレイ中に一時停止して仕切り直す手段（ポーズ/リトライ/タイトルへ戻る）もない。
ステージ選択はタイトル画面にのみ存在し、プレイ中からは戻れない。

プレイヤー体験として「音がうるさい/聞こえない」「操作が分からない」「やり直したい」「中断したい」に
応えられないため、これらを束ねたオプションメニューを用意する。

## 実装対象の機能

### 1. オプションメニュー基盤（共通UI）
- タイトル画面・ゲーム中の双方から開ける共通のオプションUIを用意する。
- 高DPI規約（`scaled()` / `scaledFontPx()`）に準拠した描画。
- 既存 `stageSelect.ts` のオーバーレイ/ボタンUIパターンと一貫した見た目。
- 効果音（`uiTap`）でフィードバックする。

### 2. 音量設定
- BGM 音量・SE 音量を個別に調整できる（0.0–1.0）。
- ミュートのオン/オフを切り替えられる。
- 変更は `SoundManager.applySettings()` で即時反映され、`SaveManager.updateSettings()` で永続化される。
- 次回起動時も設定が保持される。

### 3. ポーズ/再開
- ゲームプレイ中に一時停止し、オプションメニューを開ける。
- 再開でゲームに復帰する（ゲーム状態は保持）。
- ポーズ中はゲームの物理・入力が進行しない。
- モバイル（タッチ）でも押せるポーズ導線を用意する。
- 物理ステップ中の `scene.pause()` によるフリーズを避ける既存パターン（タイマー経由）を踏襲する。

### 4. 操作説明・キー確認
- キーボード操作（移動=矢印 / ジャンプ=SPACE / ショット=J）とタッチ操作（左半分=移動パッド、
  右側=ジャンプ/ショットボタン）を一覧表示する。
- オプションメニュー内から閲覧できる。

### 5. ステージ移動（タイトルへ戻る / リトライ / ステージ選択）
- ポーズ中から「タイトルへ戻る」でタイトル画面へ遷移できる。
- ポーズ中から「リトライ」で現在のステージを最初からやり直せる。
- ポーズ中から、または共通メニューからステージ選択を開ける（クリア済みステージへ移動可能。既存ロジックを流用）。
- 破壊的な遷移（タイトルへ戻る/リトライ）は誤操作を防げる導線にする。

## 受け入れ条件

### オプションメニュー基盤
- [ ] タイトル画面からオプションメニューを開閉できる。
- [ ] ゲーム中（ポーズ）からオプションメニューを開閉できる。
- [ ] 新規UIの絶対px/フォントサイズが `scaled()` / `scaledFontPx()` を経由している。
- [ ] 高DPI（DPR2）でレイアウトが崩れない。

### 音量設定
- [ ] BGM 音量スライダー（または段階調整）を操作すると、鳴っているBGMの音量が即時変化する。
- [ ] SE 音量を変更すると、テスト再生またはUI効果音の音量が即時変化する。
- [ ] ミュートをオンにすると全音が止まり、オフで戻る。
- [ ] 設定変更後にリロードしても値が保持されている（localStorage 永続化）。
- [ ] localStorage 不可（プライベートモード等）でもクラッシュせず動作する。

### ポーズ/再開
- [ ] プレイ中にポーズするとプレイヤー・敵・弾の動きが止まる。
- [ ] 再開するとポーズ前の状態からゲームが続く。
- [ ] ポーズ→再開を繰り返してもフリーズ・二重起動・入力の取りこぼしが起きない。
- [ ] タッチ操作のみの端末でもポーズ/再開ができる。

### 操作説明・キー確認
- [ ] キーボード操作とタッチ操作の両方が明記されている。
- [ ] 表示内容が実際の `InputController` のキーマップと一致している。

### ステージ移動
- [ ] ポーズから「タイトルへ戻る」でタイトル画面へ遷移し、BGM等が正しく切り替わる。
- [ ] ポーズから「リトライ」で現ステージが最初から再開される。
- [ ] ステージ選択から選んだステージへ正しく遷移できる。
- [ ] 破壊的遷移（タイトル/リトライ）で誤操作を防ぐ確認またはガードがある。

## 成功指標

- プレイヤーが外部知識なしに、メニューだけで音量調整・操作確認・中断/再開/やり直しを完結できる。
- 既存のゲーム進行・カットシーン・向き判定（OrientationScene）と競合しない。
- 既存テスト（lint/typecheck/unit/build）が全て通る。

## スコープ外

以下はこのフェーズでは実装しません:

- キーコンフィグ（キー割り当てのカスタマイズ）。操作は説明表示のみ。
- 言語切替・難易度設定・グラフィック品質設定。
- ゲームパッド（コントローラ）対応。
- セーブデータの手動リセット/エクスポート機能。
- 音量の数値直接入力（スライダー/段階調整で行う）。
