# 要求内容

## 概要

プレイ画面のポストFX(色調補正・ブルーム・ビネット)の画質モードを、オプションメニューから
ユーザーが変更できるようにする(AUTO / HIGH / OFF の3段階、永続化あり)。

## 背景

`src/config/graphicsQuality.ts` は当初から「将来オプションメニューからユーザー設定で
上書きできるよう、判定を 1 関数に集約しておく」と設計されており、現在は端末環境
(WebGL可否・devicePixelRatio)による自動判定のみで動いている。

- 高密度端末(生DPR>2)ではブルームが一律無効になるが、性能に余裕がある端末のユーザーが
  あえて有効にする手段がない
- 逆に低スペック端末で自動判定が有効化したFXを、ユーザーが切って60fpsを確保する手段もない

自動判定を既定としつつ、ユーザーによる上書き手段を提供する。

## 実装対象の機能

### 1. 画質モード設定(GameSettings への追加)
- `graphicsFx: 'auto' | 'high' | 'off'` を `GameSettings` に追加する(既定 `'auto'`)
- localStorage に永続化し、既存セーブは `'auto'` へ自動補完する(進捗を失わない)
- モードの意味:
  - `auto`: 現行の自動判定(WebGL かつ 生DPR≦2 でブルーム有効、色調補正・ビネットは WebGL なら有効)
  - `high`: WebGL なら全FX有効(高密度端末でもブルームを有効化)
  - `off`: 全FX無効(低スペック端末の fps 確保・素の画面を好むユーザー向け)
- Canvas レンダラでは postFX 自体が使えないため、モードに関わらず全FX無効(現行踏襲)

### 2. オプションメニューへの露出
- オプションのルートパネル(MODE / BUSTER と同列)に「エフェクト」トグルを追加する
- タップごとに AUTO → HIGH → OFF → AUTO と巡回し、即時に保存する
- タイトル・ポーズどちらから開いても表示・変更できる(既存メニューと同じ共通実装)

### 3. GameScene への反映
- `GameScene.applyPostFx()` が保存済みモードを読んで postFX を適用する
- ポーズ中にモードを変更した場合、ゲームへ戻った時点で反映される(再起動不要)

## 受け入れ条件

### 画質モード設定
- [ ] `graphicsFx` が保存・復元される(リロード後も選択が保持される)
- [ ] `graphicsFx` を持たない既存セーブが `'auto'` として読み込まれ、進捗(クリア状況・記録)が保持される
- [ ] 不正値(型・値域外)のセーブは既定値へフォールバックし、throw しない

### resolveGraphicsQuality のモード対応
- [ ] `off` で全FXが無効になる
- [ ] `high` で WebGL 時に DPR に関わらず全FX(ブルーム含む)が有効になる
- [ ] `auto`(および未指定)で現行の自動判定と同一の結果になる
- [ ] Canvas レンダラ(webgl=false)ではモードに関わらず全FX無効

### オプションメニュー
- [ ] タイトルのオプションとポーズのオプション双方にトグルが表示される
- [ ] タップで AUTO → HIGH → OFF → AUTO と巡回し、表示ラベルが追従する
- [ ] ポーズ中に変更してゲームへ戻ると、postFX の構成が変更後のモードに一致する

### 品質
- [ ] `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` が全て通る

## 成功指標

- 高密度端末(DPR>2)のユーザーが HIGH でブルームを体験できる
- 低スペック端末のユーザーが OFF で postFX 由来の描画負荷をゼロにできる

## スコープ外

以下はこのフェーズでは実装しません:

- FX単位(色調補正だけ切る等)の個別トグル
- パーティクル密度・解像度スケールなど postFX 以外の画質制御
- fps 計測に基づく動的な自動降格
