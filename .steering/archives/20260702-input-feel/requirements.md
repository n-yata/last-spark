# 要求内容

## 概要

プレイヤー操作の「手触り」を微調整する。コヨーテタイム・ジャンプ先行入力バッファ・被弾/ボス撃破時の触覚フィードバック(Android の `navigator.vibrate`)の3点を導入し、タッチ操作の気持ちよさを底上げする。

## 背景

PRD のコンセプト「手触り重視のアクション」の根幹に関わる改善。現状はジャンプ入力が「接地中の立ち上がりのみ」で受理されるため、以下の取りこぼしが起きる:

- 足場の端から一瞬遅れてジャンプを押すと不発になる(体感では「押したのに跳ばなかった」)
- 着地の直前にジャンプを押すと無視される(連続ジャンプでリズムが途切れる)

また、被弾時のフィードバックが視覚(点滅・のけぞり)と音のみで、スマホの強みである触覚を活用していない。

## 実装対象の機能

### 1. コヨーテタイム
- 足場から離れた直後の短い猶予時間(約100ms)内はジャンプ入力を受理する
- プレイヤーは足場の端ギリギリでのジャンプが安定して成立するようになる
- ジャンプで離陸した場合には適用しない(空中二段ジャンプ化しない)

### 2. ジャンプ先行入力バッファ
- 空中でジャンプ入力があった場合、短時間(約120ms)保持し、着地した瞬間に自動でジャンプを発動する
- プレイヤーは着地前の先行入力で連続ジャンプがリズムよく成立するようになる

### 3. 触覚フィードバック(バイブレーション)
- `navigator.vibrate` 対応端末(主に Android Chrome)で、以下のタイミングに短い振動を出す:
  - プレイヤー被弾時(無敵中の無効被弾では振動しない): 短い単発
  - ボス撃破時: 少し長めのパターン
- 非対応環境(iOS Safari / PC)では何もしない(エラーも出さない)
- オプションメニューで振動の ON/OFF を切り替えられ、設定は localStorage に永続化される

## 受け入れ条件

### コヨーテタイム
- [ ] 接地から離れて猶予時間内のジャンプ入力でジャンプが発動する
- [ ] 猶予時間を過ぎたジャンプ入力では発動しない
- [ ] ジャンプで離陸した直後(上昇中・下降中とも)には発動しない(二段ジャンプ不可)
- [ ] 梯子把持中は従来どおり(コヨーテ適用外)

### ジャンプ先行入力バッファ
- [ ] 着地前のバッファ時間内のジャンプ入力で、着地した瞬間にジャンプが発動する
- [ ] バッファ時間を過ぎた入力では着地してもジャンプしない
- [ ] 1回の入力で発動するジャンプは1回のみ(バッファの二重消費がない)
- [ ] 従来の「接地中に押して跳ぶ」挙動は変わらない

### 触覚フィードバック
- [ ] Android Chrome 実機で被弾時・ボス撃破時に振動する
- [ ] 無敵時間中の被弾(ダメージ無効)では振動しない
- [ ] `navigator.vibrate` 非対応環境で例外が発生しない
- [ ] オプションの振動 OFF で一切振動しなくなり、設定が再起動後も保持される
- [ ] 既存セーブデータ(振動設定キーなし)を読み込んでもエラーにならず、既定値(ON)が適用される

## 成功指標

- ジャンプ操作の不発(押したのに跳ばない)が体感上なくなる
- 既存の可変ジャンプ・梯子・ワンウェイ床の挙動が一切変わらない(既存テストが全て通る)

## スコープ外

以下はこのフェーズでは実装しません:

- 移動の加速・減速カーブの変更(現状の即時速度切替を維持)
- ショット・チャージ操作の変更
- iOS 向けの代替触覚(Vibration API 非対応のため)
- 振動の強弱・パターンの詳細カスタマイズ UI
