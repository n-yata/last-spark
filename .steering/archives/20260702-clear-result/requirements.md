# 要求内容

## 概要

リザルト画面(ClearScene)を強化する。タイムのカウントアップ演出・ベスト更新時の NEW RECORD 表示・戦績表示(被ダメージ/撃破数)・S/A/B ランク評価を追加し、ランクは永続化してステージセレクトのカードにも表示する。

## 背景

現状のクリア画面は「STAGE CLEAR + タイム」の静的テキストのみで、達成感の演出とリプレイ動機づけが弱い。ステージセレクトのカード化(20260702-stage-select-cards)でランク表示の受け皿(`StageCardModel`)を用意済みであり、リザルト強化と繋げることで「良いランクを取るためにもう一度遊ぶ」導線を作る。

## 実装対象の機能

### 1. タイムのカウントアップ演出
- クリアタイムが 0:00 から実タイムまで短時間(約900ms)でカウントアップする
- 演出中でも既存の入力受付タイミング(inputDelayMs)は変えない(進行を待たせない)

### 2. NEW RECORD 表示
- **既存のベストタイムを更新した場合のみ** NEW RECORD の強調表示を出す(初回クリアでは出さない。初回はクリア自体が祝福であり、更新の価値を薄めない)
- 判定は保存(`markStageCleared`)の**前**に既存ベストを読んで行う

### 3. 戦績表示(被ダメージ・撃破数)
- 被ダメージ(そのステージで失った HP)と撃破数(倒した雑魚敵の数)をリザルトに表示する
- GameScene がステージ中に集計し、ClearScene へ起動データで渡す
- 回復手段が存在しないため、被ダメージは「maxHp − クリア時 HP」で確定できる

### 4. S/A/B ランク評価
- 被ダメージに基づくシンプルで透明なルール:
  - **S**: 無被弾(被ダメージ 0)
  - **A**: 被ダメージが maxHp の 25% 以下
  - **B**: それ以外
- タイムはランクに含めない(タイムは BEST/NEW RECORD が既に担う。二重評価にしない)
- リザルトでランクをポップ演出付きで表示する(カウントアップ完了後)

### 5. ランクの永続化とカード表示
- ステージごとの最高ランクをセーブに記録する(S > A > B。周回をまたいで保持 = bestTimeMs と同方針)
- ステージセレクトのカードに最高ランクを表示する(BEST タイムの行の横)
- セーブ構造は現行 v7 のまま、**任意フィールド `bestRank` の追加**とする(bestTimeMs と同じ任意フィールド方式。旧セーブはフィールド無しのまま妥当であり、バージョン繰り上げ・マイグレーション不要)

## 受け入れ条件

### カウントアップ演出
- [ ] タイム表示が 0:00 → 実タイムへ増加して停止する(最終値は静的表示と同一)
- [ ] 既存の TAP TO CONTINUE / 周回選択の動作・タイミングが変わらない

### NEW RECORD
- [ ] 既存ベストより速いタイムでクリアすると NEW RECORD 表示が出る
- [ ] 初回クリア(既存ベスト無し)では出ない
- [ ] ベスト未更新(遅いタイム)では出ない

### 戦績表示
- [ ] 被ダメージと撃破数がリザルトに表示される
- [ ] 無被弾クリアで被ダメージ 0、敵を倒さずクリアで撃破数 0 が表示される
- [ ] stage3(救出演出経由)・stage6(エンディング経由)でも戦績が正しく引き継がれる

### ランク評価
- [ ] 無被弾で S、被ダメ 25% 以下で A、それ超で B が表示される(境界: ちょうど 25% は A)
- [ ] ランクがステージごとにセーブされ、より良いランクのみ上書きされる(B で再クリアしても S は残る)
- [ ] 周回リセット後もランク記録が保持される

### カード表示
- [ ] 最高ランクを持つステージのカードにランクが表示される
- [ ] ランク記録なし(既存セーブ・未クリア)ではランクが表示されず、既存表示も崩れない

### 互換性
- [ ] 既存の v7 セーブ(bestRank なし)がそのまま読み込める(進捗・設定を失わない)
- [ ] bestRank が不正値のセーブは既定値へフォールバックする(既存の破損耐性方針)

## 成功指標

- クリア時に「もう一度やってSを取りたい」と思える情報(ランク・記録更新)が一画面で伝わる
- 既存のクリア遷移フロー(全6ステージ + 周回 + hard)の回帰がない(既存テストが全て通る)

## スコープ外

以下はこのフェーズでは実装しません:

- タイムをランク基準に含めること(被ダメージのみで評価)
- ボス撃破数・ノーミス回数などの累積統計画面(別作業の「統計画面」候補)
- ランク別の報酬(解放・配色等)
- GameOver 画面の戦績表示
