# 要求内容

## 概要

ストーリー再設計の②フェーズ（各ステージのプレイ内容＝ボス・ギミック・実装後始末）をコードに反映する。物語を語る口を4つに集約する方針に伴い、科学者ログ系を全撤去し、各ボスにテーマを体現させる「安い手当て」、S4の毒床新ギミック、各ステージ構成の再配置を行う。

## 背景

①ストーリー再設計（PR#36）・②各ステージ設計（PR#38）がマージ済み。設計の正本は `docs/story.md` と `docs/ideas/last-spark-stage2-gameplay.md`。本フェーズはその設計をコード／構成に落とし込む「②実装」。③テキスト確定は本PRがマージされた後の別PRで行う（②がテキストの土台＝`logs`フィールド・`scientistLog`描画・群衆描写を壊すため、順序は②→③で固定）。

## 実装対象の機能

### 1. 科学者ログ系の全撤去
- `LogTrigger` エンティティ／GameSceneの生成・接触処理を撤去
- ログのその場表示（`storyDirector`の`logFound`/`scientistLog`変換、`StoryOverlay`の`scientistLog`暖色描画分岐）を撤去
- 型定義（`types/story.ts`の`scientistLog`/`LogSlot`/`logFound`/`StageStory.logs`）を撤去
- 全ステージconfig（`config/stage1.ts`の`logTriggers`配列・`LogTriggerSpawn`型）を撤去
- 関連テスト（`LogTrigger.test.ts`等）を撤去・修正
- ※セーブ層（`collectedLogs`）は過去に撤去済み。互換テストは残す

### 2. stage6-ending の群衆描写削除
- `config/story/cutscenes.ts` のエンディングから「施設から人間たちが姿を見せる」群衆描写ステップを削除
- 「人間どうしが争った跡（落書き・崩れたバリケード）」のト書きは story.md が「テキスト描写は可」と明示しているため**残す**

### 3. ボスの安い手当て
- **S2 哨戒機（飛行）**: 単眼／サーチライトのモチーフを figure に強調（既存の単眼センサー頭を活かす）。挙動は変えない
- **S4 環境管理機（毒）**: 毒攻撃の弾色を背景の汚染トーン（緑土気色 `#151a0c`系）と地続きの色へ変更。現状はピンク（`#ff7a90`）

### 4. S4 毒床ギミック（唯一の新規実装）
- 毒だまり（毒床）に触れている間ダメージを受ける床ハザードを新設
- `config/stage1.ts` のステージデータに毒床配置フィールドを追加し、S4に配置

### 5. 各ステージ構成の再配置（②設計の手触りを出す）
- S1: 開けて静か・敵まばら（孤独）
- S2: 既存の梯子を主役に縦攻略を強調（謎・上へ）
- S3: 足場で狭く囲んだ通路感（閉じ込め）。奥に檻
- S4: 奈落を毒床に置換／併設（揺らぎを足元で）
- S5: 足場を詰めて機械密集（冷たく硬い）
- S6: ギミック抑えめ・ボス集中（対決）
- ※既存要素（足場・梯子・奈落・背景色）の再配置が中心。新メカニクスはS4毒床のみ

## 受け入れ条件

### ログ系撤去
- [ ] `LogTrigger.ts` とそのテストが削除されている
- [ ] GameSceneにログ生成・接触処理が残っていない
- [ ] `types/story.ts` から `scientistLog`/`LogSlot`/`logFound`/`logs` が消えている
- [ ] `config/stage1.ts` から `logTriggers`/`LogTriggerSpawn` が消えている
- [ ] ゲーム起動時・各ステージプレイ時にログ関連の参照エラーが出ない
- [ ] `npm run typecheck` / `npm test` が通る

### stage6-ending 群衆削除
- [ ] エンディングカットシーンに群衆（人間たちが姿を見せる）描写がない
- [ ] 争った跡のト書きは残っている

### ボス手当て
- [ ] S2ボスに単眼／サーチライトのモチーフが視認できる
- [ ] S4ボスの毒弾が汚染トーンと地続きの色になっている
- [ ] ボスの挙動（攻撃パターン・移動）が変わっていない（回帰なし）

### S4毒床
- [ ] S4で毒床に触れ続けるとプレイヤーがダメージを受ける
- [ ] 毒床のダメージ経路が既存の被ダメージ処理と整合している
- [ ] 毒床のユニットテストがある

### ステージ構成
- [ ] 各ステージの構成が②設計の「手触り」に沿って再配置されている
- [ ] 再配置後も各ステージがクリア可能（詰み・即死配置がない）

## 成功指標

- ログ系の参照が0件（grep で `LogTrigger`/`scientistLog`/`logFound` がコードから消える）
- 既存テスト＋新規毒床テストがすべてグリーン
- lint/typecheck/test/build がすべて成功

## スコープ外

以下はこのフェーズでは実装しません:

- **③テキスト確定**（全台詞の書き直し）＝次PR（`feature/story-text-final`）
- ボスの攻撃パターン・メカニクスの全面作り替え（②方針＝安い手当てのみ）
- ルビ（ふりがな）実装
- BGM・音源の選定
- 新規の装飾オブジェクト描画（落書き・残骸・人影等の実装）
