# 要求内容

## 概要

メモリ(ログ)を拾った瞬間に流れる「科学者のログ本文(`scientistLog`)」の演出を取りやめ、後からまとめて読める専用の「ログ閲覧画面(LogViewerScene)」に隔離する。拾得時のテキスト渋滞を解消し、物語背景の情報は残す。Stage1〜6 すべてを対象とする。

## 背景

メモリ拾得時、現状は最大3系統のテキスト(`rayInner` の内心 / `scientistLog` の本文 / `eclipseVoice` の語りかけ)が連続表示される。特に最初のログでは「内心 → 科学者ログ本文 → 内心」と3連続で流れ、RAY の内心や ECLIPSE の語りかけだけでも十分なボリュームがある中で過剰(渋滞)に見える。

監督(シャビ)の判断:
- 科学者ログの物語上の重要度は「補助的・無くてもいい」レベル。
- ただし完全削除ではなく、後から読める形で残したい(隔離方針)。

## 実装対象の機能

### 1. 拾得時の演出スリム化(渋滞解消)
- メモリ拾得時に `scientistLog` 本文を流さない。
- 代わりに軽量トースト「ログを取得」を HUD 隅に短時間表示(自動消去)。
- RAY の内心(`firstLogFound` / `firstLogRead` 等)は従来どおり残す。
- ECLIPSE の語りかけ(`eclipseVoice`)はそのまま。

### 2. ログ収集状態の永続化
- どのログを取得したかを SaveData に保存(`collectedLogs: string[]`、キー形式 `"stage6:early"`)。
- SAVE_VERSION 2→3。既存セーブ(v2)を破壊しない v2→v3 マイグレーションを実装。

### 3. ログ閲覧画面(LogViewerScene)
- 取得済みログの一覧と本文を1画面で閲覧できる新規 Scene。
- 未取得ログは「???」でロック表示。
- 重厚にしすぎない(補助情報のための画面)。

### 4. 閲覧画面への入口
- タイトル画面に「記録ログ」導線を追加(DEV MODE ボタンと同じ作法)。

## 受け入れ条件

### 拾得時の演出スリム化
- [ ] メモリ拾得時に科学者ログ本文(`scientistLog`)が画面に流れない。
- [ ] 最初のログ拾得時も RAY の内心は従来どおり表示される。
- [ ] 拾得時に「ログを取得」トーストが表示され、一定時間後に自動消去される。
- [ ] Stage1〜6 すべてで上記が成立する(Stage6 の `early` 1本含む)。

### ログ収集状態の永続化
- [ ] ログ取得が SaveData の `collectedLogs` に記録され、リロード後も保持される。
- [ ] 既存の v2 セーブを読み込んでも clearedStages / bestTimeMs / settings が失われない(v2→v3 マイグレーション)。
- [ ] localStorage が利用不可・破損していても throw せず既定値で継続する(既存方針踏襲)。

### ログ閲覧画面
- [ ] タイトルから「記録ログ」を開ける。
- [ ] 取得済みログの一覧と本文が表示される。
- [ ] 未取得ログは「???」でロック表示される。
- [ ] 閲覧画面を閉じてタイトルに戻れる。

## 成功指標

- メモリ拾得時に連続表示されるテキストが「内心のみ」に減り、渋滞が解消される。
- 科学者ログの物語情報がログ閲覧画面で失われずに参照できる。

## スコープ外

以下はこのフェーズでは実装しません:
- ゲーム中(プレイ中)からログ閲覧画面を開く HUD ボタン・ポーズ機構(将来の後付けとする)。
- 科学者ログのテキスト内容そのものの変更。
- 表示パイプライン(registry→UIScene→StoryOverlay)や `storyDirector.resolveStoryEvent` 本体の改修。
- クリア画面など、タイトル以外からの閲覧導線。
