# 要求内容

## 概要

`docs/story.md` で確定したストーリーをゲーム内に表示するための「ストーリーテリング基盤」を実装する。5種のテキスト表示・ログ断片の解錠表示・ECLIPSEの語りかけ・RAYの内心テキストを仕組みとして作り、実装済みの Stage 1-2 に確定テキストを組み込む。

## 背景

Stage 1-2 はゲームプレイ（移動・戦闘・ボス）のみが実装されており、ストーリーを伝える仕組みが一切ない。`docs/story.md` には全6ステージ分の確定テキストが揃っているが、それを表示するシステムが存在しない。Stage 3以降を作る前に、まず既存の2ステージでこの基盤を確立・検証することで、以降のステージ追加が「データとテキストの組み込み」だけで済むようになる。

このブロックは全ステージの前提となる最優先ブロックである。

## 実装対象の機能

### 1. テキスト表示システム（5種の視覚的区別）

- `docs/story.md`「テキスト表示仕様（5種の視覚的区別）」に従い、5種のテキストを見た目で区別して表示する
  - 科学者のログ（画面上部オーバーレイ・手書き風・暖色）
  - ECLIPSEの語りかけ（画面上部オーバーレイ・等幅・冷色）
  - RAYの内心テキスト（画面下部・イタリック・白半透明）
  - ステージ開始テキスト（画面中央・フェードイン・白）
  - TERRAのセリフ（画面下部・標準・暖色）※基盤のみ用意（Stage 3で本格使用）
- オーバーレイ系（ログ・語りかけ）表示中はゲームを一時停止し、タップで閉じて再開
- 話者ラベル（「TERRA:」等）は表示せず、スタイルと色調で話者を区別する

### 2. ログ断片システム（RAY内蔵記録の解錠）

- ステージ内に配置した「ログトリガー（科学者の遺品・旧式端末）」にプレイヤーが接触すると、対応するログを解錠・表示する
- 触れるかどうかはプレイヤーの自由（任意接触・読み飛ばし可能）
- 配置位置は「ステージ序盤」「ボス前」「ボス後」の3種（1ステージ最大3本）

### 3. ステージ開始テキスト演出

- ステージ開始時に、画面中央へ数行のテキストをフェードイン表示する
- プレイのテンポを止めすぎない短い演出

### 4. ECLIPSEの語りかけ（ボス前演出）

- ボスエリア突入直前に、ECLIPSEの語りかけ（冷色・等幅）を一時停止つきで表示する

### 5. RAYの内心テキスト

- `docs/story.md`「RAYの内心テキストのトリガー基準」に従い、特定の瞬間に画面下部へ内心テキストを表示する（ゲームは止めない軽い表示）

### 6. Stage 1-2 への確定テキスト組み込み

- `docs/story.md`「テキストコンテンツ（確定版）」の Stage 1-2 分（ステージ開始テキスト・ログ3本ずつ・ECLIPSEの語りかけ・RAYの内心）をデータとして登録し、実際に表示されるようにする

### 7. functional-design.md の更新

- 新システム（テキスト表示・ログトリガー）を機能設計書に追記する

## 受け入れ条件

### テキスト表示システム
- [ ] 5種のテキストがそれぞれ異なる表示位置・スタイル・色調で表示される
- [ ] オーバーレイ系の表示中はゲームが一時停止し、タップで再開できる
- [ ] 話者ラベルは表示されず、見た目だけで種類が区別できる
- [ ] テキスト表示ロジックの純粋部分にユニットテストがある

### ログ断片システム
- [ ] ログトリガーに接触するとログが表示される
- [ ] 接触しなければ表示されず、ゲーム進行に影響しない
- [ ] 1ステージに最大3本（序盤・ボス前・ボス後）配置できる

### ステージ開始テキスト・語りかけ・内心
- [ ] ステージ開始時に開始テキストがフェードイン表示される
- [ ] ボス前にECLIPSEの語りかけが表示される
- [ ] 指定の瞬間にRAYの内心テキストが表示される

### Stage 1-2 組み込み
- [ ] Stage 1 で開始テキスト・ログ3本・語りかけ・内心が確定版どおり表示される
- [ ] Stage 2 で同上が表示される

### 品質
- [ ] `npm test` / `npm run lint` / `npm run typecheck` / `npm run build` が通る
- [ ] 既存の208テストが壊れていない

## 成功指標

- Stage 1-2 をプレイして、ログを拾うと「人類が囚われた世界」「科学者の存在」がテキストから読み取れる
- Stage 3以降の追加が「ステージデータ＋確定テキストの登録」だけで完結する状態になる

## スコープ外

以下はこのフェーズでは実装しません:

- 演出シーン（静止画＋交互テキスト＋ト書き）→ ブロック2
- TERRA のセリフの実使用 → ブロック2（基盤の枠だけ用意）
- Stage 3以降のステージデータ・ボス
- BGM の拡張 → ブロック6
- SaveData の6ステージ対応 → ブロック2
