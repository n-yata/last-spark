# 要求内容

## 概要

スマートフォン（高DPI / Retina）でプレイ時に描画が滲んで見える問題を解消する。PC（DPR=1）では十分鮮明だが、DPR=2〜3 の端末で canvas がアップスケールされ滲む。論理解像度を物理ピクセル化し、全UIの絶対px/fontSize を DPR 倍にスケールして鮮明化する。

## 背景

- 現状 `Phaser.Scale.RESIZE` を使用。Phaser 3.90 は `resolution` 設定が廃止され**1固定**のため、canvas のバッキングストア解像度は常に論理サイズ（CSS px）と等しい。
- DPR=2〜3 のスマホでは、CSS px の canvas を物理ピクセルへ引き伸ばすため滲む。PC（DPR=1）は等倍なので問題なし。
- 「バッキングストアだけ DPR 倍」はサポートAPIでは不可能（resolution=1 固定 / レンダラは ScaleManager から解像度を読む）。
- → 論理サイズ自体を物理px化する正攻法を採る。これに伴い、画面座標系で絶対px指定している要素（fontSize 約30箇所、タッチUI定数、配置オフセット等）は DPR 倍にスケールしないと縮むため、横断的に対応する。

## 実装対象の機能

### 1. 高DPI対応のレンダリング解像度

- canvas のバッキングストア解像度を `論理px = CSS px × cappedDpr` にする。
- canvas の CSS 表示サイズは画面いっぱい（CSS px）を維持し、レターボックスを作らない現挙動を保つ。
- DPR は GPU 負荷とのバランスから `min(devicePixelRatio, 2)` で上限キャップ。
- ウィンドウリサイズ・画面回転に追従して再適用する。

### 2. UI座標系の DPR スケール（uiScale）

- 画面座標系の絶対px（fontSize、タッチボタン半径/配置オフセット、帯高さ、不感帯、矩形サイズ、線幅等）を `uiScale(=cappedDpr)` 倍する単一ヘルパーに統一する。
- 比率指定（`width/2` 等）は `scale.width` が物理px化されることで自動スケールするため変更しない。
- これにより、見た目のサイズ・操作感は現状と完全に一致したまま、解像度だけ向上する。

## 受け入れ条件

### 高DPI対応のレンダリング解像度
- [ ] DPR=2/3 のエミュレートで canvas の `width` 属性が CSS 表示幅の DPR 倍（上限2倍）になっている
- [ ] canvas の CSS 表示サイズは画面いっぱい（従来通りレターボックスなし）
- [ ] PC（DPR=1）で従来と同一表示（回帰なし）
- [ ] 画面回転・ウィンドウリサイズ後も解像度・レイアウトが破綻しない

### UI座標系の DPR スケール
- [ ] テキスト・タッチボタン・HUD の見た目サイズが DPR に依らず現状と一致する
- [ ] タッチ操作（移動パッド原点・ジャンプ/ショット円・不感帯）の操作感が現状と一致する
- [ ] ボストリガー・縦持ち判定が現状通り機能する
- [ ] 既存ユニットテスト（touchLayout, controlBand 等）がデフォルト(uiScale=1)で不変のまま通る

## 成功指標

- スマホ実機/エミュレートでテキスト・スプライトの滲みが解消し鮮明に見える
- PC 表示・操作感に回帰がない
- スマホで 60fps を維持（DPRキャップ2で GPU 負荷を抑制）

## スコープ外

- アセット（スプライト画像）自体の高解像度化（元素材の差し替え）は行わない
- ゲームバランス・演出内容の変更は行わない
- DPRキャップ値の動的設定UI（固定で min(dpr,2)）
