# 設計: 再同期の方針

## 対象範囲の確定

コードの正本テキストが宿るのは `src/config/story/*.ts`。これと突き合わせた結果、
ズレているのは `docs/story.md` の「テキストコンテンツ（確定版）」セクションが中心。

### story.md で更新する箇所

| 箇所 | 対応するコード |
|------|----------------|
| ECLIPSEの語りかけ（確定版テーブル） | 各 stage*.ts `eclipseVoice` |
| ステージ開始テキスト（全6） | 各 stage*.ts `intro` |
| ログ断片テキスト（全6） | 各 stage*.ts `logs.early/preBoss/postBoss` |
| RAYの内心テキスト表（全6） | 各 stage*.ts `inner.*` ＋ cutscenes.ts |
| TERRAのセリフ（Stage 3〜6 台本） | cutscenes.ts の各 Cutscene.lines |
| Stage 6 結末演出の詳細構成（撃破ナレ・エンディング本文） | cutscenes.ts `stage6-ending` の narration |
| トーン指針表・Stage 3 構成の引用例 | 上記と同じ確定テキスト（内部整合のため追従） |

### 設計判断

- 確定版セクションはコードの実体テキストと **逐語一致** させる。
- トーン指針表（308行〜）と Stage 3 構成（319行〜）に旧テキストの引用が残るため、
  ドキュメント内の矛盾を避ける目的でこれらの引用も現行テキストへ追従させる。
- Stage 6 結末の内心索引（「それでも——ここから、もう一度」）は
  カットシーンに該当行が無いため、台本末尾の RAY 内心「次は——おれたちが、決める」へ揃える。
- Stage 3 台本末尾の RAY 内心「この子を、守る。…」は cutscenes.ts に含まれるため台本へ追記。
- エンディング本文の「人類」→「人間」（コードの narration に合わせる）。

### glossary.md

- ステージ名（崩れた都市/高層廃墟/収容施設/汚染地帯/ECLIPSE外縁部/ECLIPSE支配中枢）は
  既にコードのコメントと一致。平易化された画面内テキストの引用も無い。
- よって **glossary は変更不要**。固有名詞 ECLIPSE/RAY/TERRA も維持されている。

### 非対象（design レベルのため触れない）

- 世界設定・登場人物・物語の骨格などの設計プロセス文（旧来の正式文体）。
  これらは画面内テキストではなく設計記述であり、再同期の対象外とする。
