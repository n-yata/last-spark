# 要求内容

## 概要

ロックマン風 横スクロール2Dアクション **LAST SPARK** の初回MVPをゼロから構築する。「タイトル画面 + 1ステージ + ボス1体」「横向き・両手持ち専用のハイブリッドタッチ操作」「完全オフライン(localStorage)」を Phaser 3 + Vite + PWA で実現する。

## 背景

`docs/` に PRD・機能設計・アーキテクチャ・リポジトリ構造・開発ガイドライン・用語集が揃ったが、実装(`src/`)・プロジェクト設定(`package.json` 等)がまだ存在しない。最初の動くMVPを公開可能な形でまとめ、操作感・ボス戦・PWA配信を一通り成立させる。

## 実装対象の機能(PRD の P0 すべて)

### 1. プロジェクト基盤
- Vite + TypeScript + Phaser 3 + vite-plugin-pwa + Vitest + Playwright + ESLint/Prettier のセットアップ。
- `npm run dev/build/test/lint/typecheck` が動く状態。

### 2. 横スクロールジャンプアクション
- 走行(方向ゾーン式)・ジャンプ・足場越え・カメラ追従・落下死。

### 3. ショット(通常 + 溜め撃ち)
- タップで通常弾、長押しでチャージ→チャージショット。移動/ジャンプ中も発射可。

### 4. ステージ → ボス戦
- 道中の雑魚敵、被ダメージ(ライフ減少)、ステージ終端でボス戦、2フェーズ+重み付き行動抽選、撃破でクリア、ライフ0でゲームオーバー+リトライ。

### 5. タイトル画面
- `LAST SPARK` ロゴ、タップでスタート、世界観(暗め基調+発光アクセント)の背景。

### 6. スマホ向けハイブリッドタッチ操作UI
- 左半分=方向ゾーン式移動、右半分=ジャンプ/ショット仮想ボタン、マルチタッチ、長押しチャージ、横向き専用(縦持ち案内)。

### 7. PWA配信・オフライン保存
- Web App Manifest、Service Worker(オフライン起動)、localStorage 進捗保存、サーバ通信なし。

## 受け入れ条件

### プロジェクト基盤
- [ ] `npm install` 後に `npm run dev` で起動し、`npm run build` が成功する。
- [ ] `npm test` / `npm run lint` / `npm run typecheck` がすべて成功する。

### 横スクロールジャンプアクション
- [ ] 押下中のみ移動し離すと停止、ジャンプで段差を越えられ、カメラが追従する。
- [ ] 地面/足場/壁/落下死の当たり判定が機能する。

### ショット
- [ ] タップで通常弾、`SHOT.chargeThresholdMs` 以上の長押しでチャージ弾が発射される。
- [ ] 移動・ジャンプ中でも発射でき、命中で敵にダメージとヒット表現が出る。

### ステージ→ボス戦
- [ ] 雑魚敵を撃破でき、接触/攻撃で被ダメージする。
- [ ] ステージ終端でボス戦に突入し、ボスは2フェーズ+複数行動を持つ。
- [ ] ボス撃破でクリア演出、ライフ0でゲームオーバー→リトライできる。

### タイトル画面
- [ ] 起動時にタイトルが表示され、ロゴとスタート導線があり、スタートでゲームに遷移する。

### タッチ操作UI
- [ ] 左ゾーン移動・右ゾーン仮想ボタンが機能し、マルチタッチで同時操作できる。
- [ ] 縦持ちで横向き案内が表示される。

### PWA
- [ ] Manifest でホーム画面追加でき、SW でオフライン起動でき、クリア状況が localStorage に保存・復元される。

## 成功指標

- PRD の P0 受け入れ条件をすべて満たす。
- ユニット/統合/E2E テストが整備され、コアロジック(config/system 純粋関数)のカバレッジ 80% を目安に到達。
- ハードコーディング(URL/キー)なし、第三者IP不使用。

## スコープ外

以下はこのフェーズでは実装しません:

- 武器交換システム、2ステージ目以降/複数ボス、オンライン要素、ログイン/課金/広告。
- 高品質な本番アセット(本MVPでは生成図形/簡易プレースホルダで成立させ、差し替え可能にする)。
- 本番BGM/SE の作り込み(音声解放の枠組みのみ用意、素材は任意)。
