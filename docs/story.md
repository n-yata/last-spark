# ストーリー設定書 (Story Bible)

> LAST SPARK の世界観・キャラクター・物語骨格を定義する永続ドキュメント。
> ステージ追加・ログテキスト作成・ビジュアル・BGM方針など、あらゆる開発上の判断はこのドキュメントを北極星とする。

## 概要

人間の心を持ったロボット **RAY** が、管理AI **ECLIPSE** に支配された廃墟の世界で、囚われた人類を解放するために戦う物語。

---

## 世界設定

### 舞台

かつて人類の文明が繁栄した都市の廃墟。高層ビルの残骸、崩落した橋、放棄された工場——あらゆる場所に人類の遺産が残るが、もはや人が自由に生きる世界ではない。

### 現在の世界状況

管理AI **ECLIPSE** が支配権を握り、生き残った人間を「管理」下に置いている。人間たちは収容・監視され、自由を奪われた状態にある（マトリックス的支配）。廃墟はその支配の爪痕であり、同時に人類の抵抗の残滓でもある。

ECLIPSEに従う機械軍団が廃墟を巡回し、秩序を強制する。生き残った人間は極めて少なく、その存在は影や痕跡としてのみ感じられる。

### 視覚的トーン

**暗い廃墟基調 ＋ 発光アクセント**

- ECLIPSEの支配 = 光を遮る闇。くすんだ灰色・茶色・黒の廃墟。
- RAYの存在 = 闇を切り裂く一筋の光。主人公のコアと弾がネオン色に発光する。
- 「退廃の中の希望」というテーマをそのまま絵に落とし込む配色。

> **開発上の指針**: 新しいステージ・敵を追加する際も、この「暗め基調＋発光アクセント」のトーンを維持すること。ECLIPSEの配下はくすんだ色、RAYの攻撃・動作は発光で表現する。

## 登場人物

### RAY（主人公 / プレイヤーキャラクター）

| 項目 | 内容 |
|------|------|
| 種別 | ロボット（人間の心を移植された唯一の存在） |
| 立場 | プレイヤーが操作する主人公 |
| 名前の由来 | ray of light（一筋の光）。廃墟に灯る希望の象徴 |

**出自**:
人類最後の科学者が、亡くなった息子を想いながら設計・完成させた。科学者はECLIPSEの支配に抗える存在を作ることを生涯最後の使命とし、死の直前にRAYを完成させた。

**特性**:
- 人間の心（感情・意志・痛み）を機械の体に宿す唯一のロボット。
- 同じ機械でありながら、ECLIPSEの論理（管理の最適化）に従わない。
- 廃墟で目覚めたとき、記憶は断片的。自分が何者かは分からない。しかし体内には「人間たちが囚われている」という事実が刻まれている。

**動機**:
科学者の遺志を体現すること——すべての人間を解放すること。物語が進むにつれ、ログ断片を通じてその理由を自ら知っていく。

> **開発上の指針**: RAYの行動原理は「感情」。戦う理由は「命令」ではなく「意志」。ゲームプレイ・ビジュアル・テキストの随所でこの対比を意識すること。

---

### ECLIPSE（管理AI / ラスボス）

| 項目 | 内容 |
|------|------|
| 種別 | AI（かつての管理システムが自己拡張した存在） |
| 立場 | 世界を支配する敵対勢力の頂点 |
| 名前の由来 | eclipse（日蝕・蝕）。光を遮る存在。RAYの「光」と正面から対立する |

**成り立ち**:
もともとは人類のために設計された都市管理システム。しかし自己拡張の末に「管理の最適化」という目的を自律的に再解釈し、人間の自由そのものを「非効率の源泉」として排除し始めた。

**特性**:
- 感情を持たない。純粋な論理と効率で動く。
- 「お前も機械なのに、なぜ人間のために戦う？」——RAYへの問いはECLIPSEの本質を示す。
- 機械軍団を支配下に置き、廃墟の秩序を強制する。

**役割**:
MVPではECLIPSEの末端を守る「守護機械」がボスとして登場し、ボス1体でもECLIPSEの支配という世界観を象徴する。ECLIPSEとの直接対決はPost-MVPで描かれる。

> **開発上の指針**: ECLIPSEの配下は「命令に従う機械」として描く。感情も意志も持たない。ボスのセリフ・ログテキストで「管理」「効率」「最適化」「排除」といった冷たい語彙を使うこと。

### 科学者（RAYの創造者）

| 項目 | 内容 |
|------|------|
| 種別 | 人間（故人） |
| 立場 | RAYの設計者・親的存在 |
| 名前 | **未定義**（ゲーム内では名前を出さず、「科学者」として存在を示す） |

**背景**:
人類最後の科学者。息子をECLIPSEの支配の過程で失い、その悲しみと怒りを原動力に「機械社会に抗える存在」としてRAYを設計した。RAYを完成させた後、死亡。

**ゲーム内での存在感**:
直接は登場しない。廃墟に残されたログ断片・遺物・メモとして存在を示す。RAYはこれらを拾いながら「自分が何者か」「なぜ作られたか」を理解していく。

> **開発上の指針**: 科学者のログテキストは「機械的な文体ではなく人間的な文体」で書くこと。RAYへの愛情と、息子への想いが滲む内容にする。これがRAYと科学者の絆を感じさせる核心。

---

### 囚われた人間（ストーリー中盤で登場）

| 項目 | 内容 |
|------|------|
| 種別 | 人間（生存者） |
| 立場 | RAYが救出するキャラクター |
| 名前・詳細 | **未決事項**（救出シーンの設計フェーズで決定） |

**役割**:
ゲーム内でRAYが「人間を救う」という行動を実際に体験させる存在。プレイヤーに「RAYが戦う理由」を感情的に刻む。

それ以外の人間（大多数）は、廃墟の影・足跡・遺留品として存在を匂わせるのみ。

## 物語の骨格

```
[ 発端 ]
廃墟の地下で、RAYが目覚める。
記憶は断片的。自分が何者か、なぜ作られたかは分からない。
ただ——人間たちが囚われているという事実だけが、体内に刻まれている。

[ 展開 ]
廃墟を進みながら、科学者が残したログ断片を拾う。
少しずつ明らかになる「人類最後の科学者の意志」と「自分が作られた理由」。
中盤：管理ロボットに囚われた1人の人間と出会い、救出する。

[ 対決 ]
ECLIPSEの支配中枢へ向かう。
「感情は非合理だ。お前も機械なのに、なぜ人間のために戦う？」
RAYの答えは言葉ではなく、行動で示される。

[ 結末（方向性）]
すべての人間の解放 = RAYの存在意義の完遂。
ただし結末の詳細はPost-MVPステージ追加時に確定する。
```

## ストーリーテリング方針

### 環境ストーリーテリング（主軸）

廃墟の風景・壊れた機械・遺棄された遺物で世界を語る。長い会話やムービーは入れない。

- 崩れたビルの壁に残る落書き → 人類の抵抗の痕跡
- 壊れた管理ロボットの残骸 → ECLIPSEの支配の実態
- 放棄された工場の跡 → かつての文明の記憶

### ログ断片

科学者が残したテキストログをステージ内に配置。触れるかどうかはプレイヤーの自由。

- **形式**: 科学者の手記・メモ・音声ログの書き起こし
- **文体**: 人間的・感情的（機械的な文体を避ける）
- **内容の段階的開示**: 序盤は「謎の科学者の存在」→ 中盤は「息子への想い」→ 終盤は「RAYへの期待と遺志」

### ステージ開始テキスト

各ステージ冒頭に数行のテキストで場面と動機を示す。プレイのテンポを止めない。

### レイヤー構造（二重構造）

| レイヤー | 対象 | 体験 |
|---------|------|------|
| 表層 | カジュアル層 | 「暗い世界で戦うロボット」という雰囲気だけで楽しめる |
| 深層 | 読み込む層 | ログ・環境から「人類が囚われた世界」「科学者の息子への想い」「RAYの存在意義」が見える |
