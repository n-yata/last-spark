# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**LAST SPARK**(ラストスパーク) - スマホで遊ぶ、ロックマン風 横スクロール2Dアクション

### プロダクトコンセプト
- **退廃の中の希望**: 管理AI **ECLIPSE** に支配された廃墟の世界で目覚めた「最後のロボット **RAY**」が、人間の心を持つ唯一の機械として支配に抗い、人類を解放するために戦う。暗め基調の廃墟に発光アクセントを灯すアートで世界観を体現する。詳細なストーリー設定は `docs/story.md` を参照。
- **手触り重視のアクション**: 走る・ジャンプ・足場越え・通常ショット・チャージショットという「ロックマンらしさのコア」を、スマホのタッチ操作で気持ちよく成立させる。
- **スキマ時間で遊べるPWA**: インストール不要・サーバ不要。1ステージ数分で完結し、ホーム画面に追加して手軽に起動・プレイできる完全オフライン体験。

### プロダクトビジョン
LAST SPARK は、レトロ風2Dアクションを愛するカジュアルゲーマーが、スマホのスキマ時間に「退廃の中の希望」という世界観を数分で味わえる横スクロールアクションゲームである。
高難度を競うのではなく、タッチ操作の気持ちよさとボス戦の達成感、そして環境ストーリーテリングによる「先が気になる」引きを提供する。
全 6 ステージ + 全 6 ボスでストーリーを完結させ、周回(New Game+)・難易度切替・強化バスターモードでリプレイ性を持たせた完成形として公開済み。以降は運用フェーズとして、バランス調整・演出強化・新規コンテンツ追加を段階的に行う。

### 目的
- ロックマン風の横スクロール2Dアクションを **PWA として公開し、実際に遊んでもらう**。
- スマホのタッチ操作で、走る・ジャンプ・ショット・チャージショットの手触りを成立させる。
- 全 6 ステージ + 全 6 ボスの構成で、操作感・ボス戦・PWA 配信を完成度高くまとめる。
- 周回・難易度・強化バスターモードにより、1周後も遊び続けられるリプレイ性を提供する。
- 権利侵害のないオリジナル設計(キャラ・スプライト・名称・音楽すべて独自)で安心して公開する。

## 成功指標(KPI)

> 完全オフライン(端末内のみ)・サーバ不要のため、サーバ集計に依存しない指標を主とする。計測は端末内 localStorage の任意記録、もしくは公開後に必要に応じて導入する軽量アクセス解析(プライバシー配慮の上で検討)を想定する。現時点ではサーバ送信を行わない。

### プライマリーKPI
- **全ステージクリア率(初回到達者ベース)**: ステージ1を開始したプレイヤーのうち、全6ステージ(ECLIPSE本体)を撃破するまで到達する割合 40% 以上(公開後3ヶ月時点、端末内計測の任意提供データで評価)。
  - 「難しすぎず歯ごたえはある」難易度方針の妥当性を測る。
- **チュートリアルなしでの操作習熟**: 初見プレイヤーがステージ1最初の足場区間(ジャンプ＋ショットを要する導入区間)を 90 秒以内に通過できる(ユーザビリティテスト 5 名以上で測定)。

### セカンダリーKPI
- **PWA ホーム画面追加率**: プレイしたユーザーのうちホーム画面に追加する割合 15% 以上(公開後3ヶ月、計測可能な場合)。
- **平均1プレイ時間**: 1 ステージあたり 3〜7 分に収まる(「スキマ時間で数分」の設計目標)。全6ステージ通しでは 20〜40 分程度を想定。
- **周回(New Game+)到達率**: 全ステージクリア済みプレイヤーのうち、次の周回へ進む割合 20% 以上(リプレイ性の指標)。
- **クラッシュ・致命的不具合ゼロ**: 対応端末/ブラウザでプレイ続行不能になる不具合の報告ゼロ(公開後1ヶ月)。

## 機能要件

### コア機能

#### 1. 横スクロールジャンプアクション

**ユーザーストーリー**:
カジュアルゲーマーとして、レトロな2Dアクションの手触りを味わうために、走る・ジャンプ・足場越えで進める横スクロールステージが欲しい。

**受け入れ条件**:
- [ ] 主人公が左右に歩行/走行でき、押している間だけ移動し、離すと停止する(方向ゾーン式)。
- [ ] ジャンプができ、高低差のある足場(段差・ギャップ)を越えられる。
- [ ] ステージは主人公の進行に合わせて横スクロールする(カメラ追従)。
- [ ] 足場の端で踏みとどまれる(精密な移動が成立する)。
- [ ] 地面・足場・壁・落下死(または奈落)の当たり判定が機能する。

**優先度**: P0(必須)

#### 2. ショット(通常 + 溜め撃ち)

**ユーザーストーリー**:
プレイヤーとして、手応えのある攻撃の手触りを得るために、タップでチャージを溜めて撃て、長押しでは通常弾を連射できる仕組みが欲しい。

**受け入れ条件**:
- [ ] ショットボタンの 1 回目タップでチャージが始まり、視覚的にチャージ状態が分かる(指を離してもゲージが蓄積する)。
- [ ] 一定以上チャージした状態でもう一度タップすると、チャージショット(通常より強い/大きい弾)が発射される(チャージ未成立なら通常弾)。
- [ ] ショットボタンを長押しすると、チャージせず通常弾を連射する(数発ごとに小休止を挟む)。
- [ ] 移動中・ジャンプ中でもショットを撃てる。
- [ ] 弾は敵にダメージを与え、命中時にヒット表現が出る。

**優先度**: P0(必須)

#### 3. ステージ → ボス戦(全6ステージ)

**ユーザーストーリー**:
プレイヤーとして、達成感を得るために、雑魚敵を倒しながらステージを進み、最後に行動パターンを持つボスと対決したい。

**受け入れ条件**:
- [ ] ステージ道中に雑魚敵が配置され、ショットで撃破できる。
- [ ] 雑魚敵・ボスの攻撃や接触で主人公がダメージを受ける(ライフ減少)。
- [ ] ステージ終端でボス戦に突入する。
- [ ] ボスは複数の行動パターン(移動/攻撃)を持ち、単調でない。
- [ ] ボスを撃破するとクリア状態になり、クリア演出(被ダメ基準の S/A/B ランク評価込み)が表示される。
- [ ] 主人公のライフが尽きるとゲームオーバーとなり、リトライできる。
- [ ] 全6ステージが実装され、各ステージ固有のボスと対決できる:
  1. ウンブラ(UMBRA・接地型・守護機械) — stage1
  2. コロナ(CORONA・飛行型・哨戒機) — stage2
  3. ヴェイル(VEIL・重装型・収容番人) — stage3
  4. ミアズマ(MIASMA・浄化型・環境管理機) — stage4
  5. ヘラルド(HERALD・高速飛行型・使者) — stage5
  6. エクリプス(ECLIPSE・管理AI本体・ラスボス、複数フェーズ) — stage6(最終ステージ、撃破後はエンディング演出)
- [ ] hard 難易度では stage6 のエクリプス撃破後に裏ボス(高速接近型)と連戦できる。

**優先度**: P0(必須)

#### 3b. ステージ選択・周回・難易度

**ユーザーストーリー**:
プレイヤーとして、自分のペースで遊び返すために、クリア済みステージを選んで再挑戦したり、全クリア後により歯ごたえのある難易度で周回したりしたい。

**受け入れ条件**:
- [ ] タイトル画面からカード式のステージ選択画面に遷移でき、クリア済み・未解放ステージが視覚的に区別される。
- [ ] ステージごとに最速クリアタイムと最高ランク(S/A/B)が記録され、選択画面で確認できる。
- [ ] 難易度を normal / hard から選択できる。hard は被ダメージ・敵の強さが強化される。
- [ ] 全ステージクリア後、「次の周回へ」で周回数(New Game+)を進めてクリア状況がリセットされ、最速タイム・最高ランクは保持されたまま再挑戦できる。
- [ ] 強化バスターモード(オプションのON/OFFトグル)を有効にすると、通常弾の2発化とチャージビームが全ステージで使用できる。

**優先度**: P0(必須・公開済み機能)

#### 4. タイトル画面

**ユーザーストーリー**:
初めて訪れたプレイヤーとして、世界観に惹き込まれてプレイを始めるために、起動時にゲーム名とスタート導線が示されたタイトル画面が欲しい。

**受け入れ条件**:
- [ ] 起動時に最初にタイトル画面が表示される。
- [ ] ゲーム名 `LAST SPARK` のロゴが表示される。
- [ ] 「タップでスタート」など明確なスタート導線がある。
- [ ] 背景は世界観(暗め基調＋発光アクセント)を反映している。
- [ ] スタート操作でステージ(またはステージ開始の導入)に遷移する。

**優先度**: P0(必須)

#### 5. スマホ向けタッチ操作UI(ハイブリッド操作)

**ユーザーストーリー**:
スマホを横持ちで両手で構えるユーザーとして、左右の親指で精密に操作するために、画面分割によるタッチ操作UIが欲しい。

**受け入れ条件**:
- [ ] スマホ横向き(ランドスケープ)を前提とし、両手で端末を構えて左右の親指で操作する設計である。
- [ ] 画面左半分(左手親指)が左右移動(方向ゾーン式: 押し続けている間その向きに歩き、離すと止まる)として機能する。
- [ ] 画面右側(右手親指)にジャンプボタンとショットボタンが配置される。
- [ ] ジャンプボタンで跳ぶ(押している長さで高さが変わる可変ジャンプ)。
- [ ] ショットボタンはタップでチャージ→再タップで発射、長押しで通常弾を連射する。
- [ ] 移動(左親指)とジャンプ/ショット(右親指)を同時に操作できる(指の競合がない)。
- [ ] 仮想ボタンは親指で隠れにくく、押しやすい位置・サイズである(実機調整を前提とする)。

**優先度**: P0(必須)

#### 6. PWA 配信・オフライン保存

**ユーザーストーリー**:
ユーザーとして、手軽に起動し続きから遊ぶために、ブラウザからインストールでき、サーバなしで進捗が保存されるPWAが欲しい。

**受け入れ条件**:
- [ ] Web App Manifest により、ホーム画面に追加できる(アプリ名 `LAST SPARK`、アイコン、スプラッシュ)。
- [ ] Service Worker により、オフラインでも起動・プレイできる。
- [ ] スコア/クリア状況などの進捗が localStorage に保存され、再訪時に参照できる。
- [ ] サーバ通信を行わない(完全オフライン、端末内のみで完結)。

**優先度**: P0(必須)

#### 7. オプション設定

**ユーザーストーリー**:
プレイヤーとして、自分の端末・好みに合わせて遊ぶために、音量・触覚フィードバック・グラフィック品質を調整できるオプション画面が欲しい。

**受け入れ条件**:
- [ ] BGM/SE 音量とミュートを個別に設定できる。
- [ ] 触覚フィードバック(Vibration API)のON/OFFを切り替えられる(非対応環境では設定に関わらず無効)。
- [ ] グラフィック品質(ポストFX: 色調補正/ブルーム/ビネット)を AUTO(端末の描画方式・画面密度から自動判定) / HIGH(常時全FX有効) / OFF(全FX無効) から選べる。
- [ ] 設定は localStorage に保存され、再訪時も引き継がれる。

**優先度**: P1(重要)

### タッチ操作仕様

**前提: スマホ横向き(ランドスケープ)・両手持ち専用。** 端末を両手で構え、左右の親指で操作する。

```
（スマホ横向き / 両手持ち）
┌───────────────────────────┬───────────────────────────┐
│   左半分(移動ゾーン)       │      右半分(アクション)     │
│   ＝左手 親指              │      ＝右手 親指            │
│                           │           [ジャンプ]       │
│   ◀  歩行  ▶              │   [ショット]               │
│   (押している間その向きへ) │  (タップ=溜め/長押し=連射) │
│   (離すと停止)            │                            │
└───────────────────────────┴───────────────────────────┘
```

- 画面向き: 横向き専用(縦持ちでは回転を促す案内を表示する想定)。
- 左半分(左手親指): 追従パッド。横変位で「左/右の押し分け」移動(移動専用)。
- 右側(右手親指): ジャンプボタン(右上)とショットボタン(左下)。ジャンプは押している長さで高さが変わる可変ジャンプ。ショットはタップでチャージ→再タップで発射、長押しで通常弾を連射(数発ごとに小休止)。
- マルチタッチ: 移動(左親指)とジャンプ/ショット(右親指)を同時入力可能。別ポインタに分離したため指が競合しない。
- 具体的なレイアウト・サイズ・透明度は実機で調整する(設計フェーズで詰める)。

### 将来的な機能(公開後の拡張候補)

#### ボスごとの弱点武器システム
ボス撃破で新武器を入手し、他のボスに有利を取れる「ロックマンらしい」弱点システム。現状は強化バスターモード(オプションのON/OFF)による全体強化に留まり、ボスごとに個別武器を使い分ける仕組みは未導入。複数ボスの弱点バランス設計が重いため、必要性を見極めた上で検討する。

**優先度**: P2(できれば)

#### ステージ追加(7以降)
新ロケーション・新ボスの追加。全6ステージでストーリーは完結しているため、追加する場合は本編後日談・番外編としての位置付けを検討する。

**優先度**: P2(できれば、継続コンテンツ拡充の候補)

#### オンライン要素(ランキング等)
スコアランキングやリーダーボード。サーバ運用コスト・セキュリティリスクを伴うため、人気が出てからの後付け候補。

**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス
- ミドルレンジのスマートフォン実機で、ゲームプレイ中のフレームレートが平均 60fps を維持する(具体的な下限値は運用フェーズで計測)。
- 2回目以降の起動(Service Worker キャッシュ利用)で、タイトル画面が高速に表示される(具体的な目標時間は運用フェーズで計測)。
- 入力(タッチ)からキャラクター反応までの遅延が 1〜2フレーム以内(60fps 換算で約 16〜33ms)に収まる。
- 高密度端末(生 devicePixelRatio が高い機種)でも 60fps を維持できるよう、グラフィック品質(AUTO/HIGH/OFF)でポストFXの負荷を調整できる。

> 起動時間とフレームレート下限の確定目標値は現時点では未設定。基準端末の選定と計測手順の確定を含め、運用フェーズで計測ガイドとして整備する(`docs/architecture.md`「パフォーマンス要件」参照)。

### ユーザビリティ
- チュートリアル無しでも、初見プレイヤーが導入区間で基本操作(移動・ジャンプ・ショット)を直感的に理解できる。
- 仮想ボタンが親指で隠れにくく、誤操作しにくいレイアウトである。
- 画面向きは横持ち(ランドスケープ)専用とし、両手持ちでの操作を前提とする。縦持ち時は横向きへの回転を促す案内を表示する。
- 難易度は「難しすぎず、でも歯ごたえはある」調整(本家ロックマン並みの高難度は狙わない)。

### 信頼性
- ゲームの進捗(クリア状況等)が localStorage に保存され、再訪時に失われない。
- 想定内のプレイ操作でクラッシュ・操作不能が発生しない。
- localStorage が利用不可/クリアされた場合でも、ゲーム自体は起動・プレイできる(進捗が無いだけ)。

### セキュリティ
- サーバ通信を行わない(完全オフライン)ため、外部送信による情報漏洩リスクを持たない。
- API エンドポイント・キー等の機密情報をソースコードに持たない(外部通信なし)。
- 第三者の知的財産(キャラ・スプライト・名称・音楽)を使用しない。すべてオリジナル、もしくは商用利用可能なライセンス(CC0 等)の素材のみを使用する。

### スケーラビリティ(拡張性)
- ステージデータ・敵配置・ボス行動パターンを、新ステージ/新ボスを追加しやすい構造で持つ(`config/stages.ts` にコード定義。詳細は `docs/architecture.md` 参照)。
- ショット種別(`ProjectileKind`)は拡張可能な形で設計されており、通常/チャージに加えてミサイル・槍型など複数種別、および `Beam`(強化バスターのチャージビーム)を追加済み。将来のボス固有弱点武器システム導入の土台として活用できる。

### 互換性
- 対応端末/ブラウザ: モバイルの主要ブラウザ(iOS Safari / Android Chrome)を主対象とする。iOS Safari の PWA/オーディオ制約を考慮する。具体的な対応範囲は設計フェーズで確定する。

## スコープ外

明示的にスコープ外とする項目:
- **ボスごとの弱点武器システム**(撃破武器を入れ替えて弱点を突く仕組み。複数ボスのバランス設計が重いため見送り。現状は強化バスターモードによる全体強化のみ)。
