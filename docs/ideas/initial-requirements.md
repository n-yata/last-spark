# 初期要求

> 壁打ち(grill-with-docs)による下書き。確定した内容から都度追記している。未確定の論点は末尾の「未解決の論点」に残す。

## 解決したい課題

- スマホで手軽に遊べる、ロックマン風の横スクロール2Dアクションゲームを作り、**公開して実際に遊んでもらう**こと。
- 目的は「公開して遊んでもらう」なので、操作感(タッチ操作)とゲームとしての完成度が重要になる。

## タイトル

- **タイトル名: `LAST SPARK`**(ラストスパーク)。
- 由来: 心を残した最後の機械=「退廃の中の唯一の灯(spark)」。テーマ「退廃の中の希望」をそのまま名前に込めた。
- 短く発音しやすく、PWAアプリ名/アイコン/スプラッシュにも収まりが良い。既存作品の名称・固有名は使わないオリジナル名。

## 世界観・コンセプト

> ゲームの心臓部。トーン・主人公・敵・ビジュアル・BGMはすべてここから派生する。

### 舞台とトーン
- **ディストピア / 退廃的な世界**。崩壊した文明、廃墟、機械の墓場。
- ビジュアルはシリアスで雰囲気重視。ただしターゲット(カジュアル層)に合わせ、**物語はシンプルに飲み込める重さ**に調整する(暗すぎ・難解すぎを避ける)。

### 主人公
- **滅びた世界で目覚めた「最後のロボット(旧時代の機械生命)」**。
- 孤独な機械が、なぜ世界は滅びたのか——その謎と再生の鍵を求めて廃墟を進む。
- 退廃世界 × 孤独な機械主人公でトーンが一致。世界の真相をステージ進行で少しずつ明かし、カジュアル層でも「先が気になる」引きを作る。

### 敵・対立軸
- **暴走した管理AI / 機械軍団**。かつて世界を管理していたAIが暴走し、機械が文明を滅ぼした。
- 構図は **「狂った機械(管理AI) vs 心を残した機械(主人公)」**。同じ機械でありながら支配に抗う対比がテーマの核。
- MVPのボスは、この管理AIの末端を守る「守護機械」に据えると、ボス1体でも世界観を背負わせられる(案)。

### ストーリーの見せ方
- **環境ストーリーテリング中心 + 最小限のテキスト**。
- 長い会話やムービーは入れない。廃墟の風景、道中に落ちたログ断片、ステージ開始時の数行で世界を匂わせる。
- 「読みたい人だけ深く読める」レイヤー構造。カジュアル層の遊びのテンポを止めない。

### ビジュアル / アートの方向性
- **暗めの基調 + 一点の鮮やかな発光アクセント**。沈んだ廃墟色の中で、主人公の機械や敵のコアがネオン色に発光する。
- 「退廃の中の希望」というテーマ・主人公像をそのまま絵に落とし込む。
- レトロなドット絵(ピクセルアート)と相性のよい配色。発光表現はシンプルなグロー/明度差で表現可能。

### MVP の舞台(1ステージ + ボス1体の具体化)
- **ロケーション: 崩れた都市 / 高層廃墟**。人類の遺産が残る、文明崩壊後の街。
  - 高低差のある足場、崩落した建物、瓦礫の中をスクロールして進む。横スクロールジャンプアクションと噛み合う地形。
  - 暗い街並みの中で、主人公や敵のコアが発光する「暗め基調＋発光アクセント」の絵作りを最初のステージから提示できる。
- **ボス: 街を徘徊する大型警備機**(管理AIの末端として街を"管理"し続ける防衛機械)。1体で世界観(暴走AIの支配)を象徴させる。
- 導入の動機: 目覚めた最後のロボットが、滅びの謎と再生の鍵を求めて崩れた街を進み、その地を支配する警備機と対峙する。

## ターゲットユーザー

- **レトロ風2Dアクションを好むカジュアルゲーマー**。スマホでスキマ時間に短時間(1ステージ数分)プレイする層。
- 難易度方針: 難しすぎず、でも歯ごたえはある調整。本家ロックマン並みの高難度は狙わない。
- この層を軸に、プレイ時間の目安(1ステージ数分)・操作のとっつきやすさを設計判断の基準とする。

## 主要機能

MVPで再現するロックマンらしさのコア:

1. **横スクロールジャンプアクション** — 走る・ジャンプ・足場越え。スクロールするステージ。
2. **ショット(通常＋溜め撃ち)** — ボタンでショット、長押しでチャージショット。手触りの核。
3. **ステージ→ボス戦** — 雑魚を進んで最後にボス。ボスは行動パターンを持つ。

## MVP の範囲

### 作る
- 上記の主要機能3つ
- **タイトル画面**(起動時に最初に表示。ゲーム名 `LAST SPARK` 表示＋スタート操作)。
  - PWAで公開して遊んでもらうため、第一印象となるタイトル画面は必須。
  - 構成要素(案): ロゴ表示、「タップでスタート」導線、世界観を匂わせる暗め基調＋発光アクセントの背景。
  - 詳細レイアウト・演出・BGM/SEは PRD/設計フェーズで詰める。
- **ボリューム: 1ステージ + ボス1体**(操作感・ボス戦・PWA配信を一通り完成させる最小構成。公開後にステージを追加していく)
- スマホ(PWA)向けのタッチ操作UI
  - **操作方式(ハイブリッド)**: 画面左半分=左右移動、画面右半分=仮想ボタン(ジャンプ＋ショット)。
  - 左半分の移動は **方向ゾーン式(押し続けている間その向きに歩き、離すと止まる)**。見た目はスワイプ風でも内部的には「左右の押し分け」として扱い、足場上での精密な移動・踏みとどまりを成立させる。
  - チャージショット = ショットボタン長押し。

### 作らない(MVPでは見送り)
- **武器交換システム**(ボス撃破で新武器入手→他ボスに有利)。複数ボスのバランス設計が重いため、MVP後の拡張候補とする。
- **オンライン要素**(サーバ連携、ランキング等)。MVPは**完全オフライン(端末内のみ)**。スコア/クリア状況は localStorage に保存。サーバ不要で運用コスト・セキュリティリスクを抑える。ランキング等は人気が出てからの後付け候補。

## 制約・前提

- プラットフォーム: スマートフォン向け **PWA**(ブラウザで動作、ホーム画面追加可能)。
- 技術スタック: TypeScript 5.x / Node.js v24 / npm(devcontainer)。
- **ゲームエンジン: Phaser 3**(2Dゲーム定番。スプライト/アニメ/当たり判定/タイルマップ/サウンドを内包。TypeScript対応)。
- ビルド/開発サーバ: (検討中だが Vite + PWA プラグインが有力)。
- **アセット調達: フリー素材(CC0等、Kenney / OpenGameArt 等)中心 + 不足分を生成AI/自作で補完**。
- **⚠️ 権利上の前提**: 「ロックマン」はカプコンの商標・著作物。公開するため、キャラクター・スプライト・名称・音楽はすべて**オリジナル(ロックマン"風"の独自設計)**とし、既存作品の素材・名称・固有名は使用しない。

## 未解決の論点(PRD/設計フェーズで詰める)

- ボスの具体的な行動パターン・攻撃の種類
- ステージのギミック(トゲ、移動床、消える足場 等)の採否
- ライフ/残機/コンティニューの仕様
- ビルド構成の確定(Vite + vite-plugin-pwa 想定)とPWA要件(Service Worker, manifest, アイコン)
- 対応端末/ブラウザの範囲(iOS Safari の制約含む)、画面向き(横持ち/縦持ち)
- サウンドの有無とミュート対応(モバイルの自動再生制約)
